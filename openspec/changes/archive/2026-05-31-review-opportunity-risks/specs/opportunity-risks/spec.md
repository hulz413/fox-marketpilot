## ADDED Requirements

### Requirement: 系统为每个商机生成风险复核

系统 SHALL 在研究任务已有商机结果后，为每个当前未软删除商机生成一条风险复核，用于提示用户哪些风险会影响小预算验证决策。

#### Scenario: 为已完成分析的任务生成风险复核

- **WHEN** 研究运行已经保存商机结果、货源候选、竞品参考和验证预算估算
- **THEN** 系统为每个当前商机生成风险复核
- **AND** 每条风险复核包含综合风险等级、风险摘要、分项风险说明、风险触发原因和缓解建议
- **AND** 每条风险复核关联对应研究任务和商机

#### Scenario: 上游增强数据不足时仍生成谨慎风险提示

- **WHEN** 某个商机缺少来源线索、货源候选、竞品参考或验证预算估算
- **THEN** 系统仍可以基于任务输入、商机字段和已有增强数据生成待验证风险提示
- **AND** 系统将风险复核标记为 `fallback` 或 `insufficient_data`
- **AND** 风险摘要必须提示需要继续确认供应商、平台规则、履约或售后等关键不确定性

#### Scenario: 本地或测试环境使用确定性回退

- **WHEN** 系统运行在 local 或 test 环境且未配置 LLM API key
- **THEN** 系统使用确定性 fallback 生成每个商机的风险复核
- **AND** fallback 结果仍满足风险复核 schema
- **AND** 自动化测试不依赖外部 LLM、搜索或网络服务

### Requirement: 系统持久化风险复核

系统 SHALL 将风险复核保存为可读取的结构化结果，并在重新运行时避免旧风险复核污染当前结果。

#### Scenario: 保存风险复核字段

- **WHEN** 系统成功生成风险复核
- **THEN** 系统保存每条风险复核的公开 UUID
- **AND** 系统保存研究任务关联和商机关联
- **AND** 系统保存综合风险等级、风险摘要、产品质量风险、发货履约风险、售后风险、合规风险、库存积压风险、同质化竞争风险、平台规则风险、风险触发原因、缓解建议和复核状态
- **AND** 系统保存创建时间、更新时间和可为空的删除时间

#### Scenario: 重新运行替换旧风险复核

- **WHEN** 一条已完成研究任务被重新运行并成功生成新商机结果和新风险复核
- **THEN** 系统将该任务上一轮风险复核从默认读取中排除
- **AND** 系统保存新的风险复核
- **AND** 后续读取该任务风险复核列表时默认返回最新结果

#### Scenario: 默认读取不返回软删除风险复核

- **WHEN** 用户读取某条研究任务或某个商机的风险复核
- **THEN** 系统默认只返回 `deleted_at` 为空的风险复核记录
- **AND** 系统不返回该任务旧运行中已被替换的软删除风险复核

### Requirement: 用户可以读取风险复核

系统 SHALL 提供按研究任务和按商机读取风险复核的 API，用于商机详情和基础报告展示。

#### Scenario: 读取任务下全部风险复核

- **WHEN** 用户请求一条存在且未软删除研究任务的风险复核列表
- **THEN** 系统返回该任务当前未软删除商机的风险复核
- **AND** 结果可按商机推荐排序或风险复核记录创建顺序稳定展示
- **AND** API 响应不暴露研究任务、商机或风险复核的内部自增 ID

#### Scenario: 读取单个商机的风险复核

- **WHEN** 用户请求一个存在且未软删除商机的风险复核
- **THEN** 系统返回该商机当前风险复核
- **AND** 响应包含关联研究任务 UUID 和商机 UUID
- **AND** API 响应不暴露内部自增 ID

#### Scenario: 尚无风险复核时返回空结果

- **WHEN** 用户请求一条尚未生成风险复核的任务或商机
- **THEN** 系统返回空列表或空结果
- **AND** API 响应保持成功
- **AND** 前端可以展示中文非阻塞空状态

### Requirement: 前端展示风险复核

前端 SHALL 在商机详情和基础报告中展示风险复核，让用户理解哪些风险需要在投入首批预算前优先排查。

#### Scenario: 商机详情展示完整风险复核

- **WHEN** 用户打开一个已有风险复核的商机详情页
- **THEN** 页面展示综合风险等级、风险摘要、产品质量风险、发货履约风险、售后风险、合规风险、库存积压风险、同质化竞争风险和平台规则风险
- **AND** 页面展示风险触发原因和缓解建议
- **AND** 页面使用“风险提示”“待验证”“需要确认”或等价谨慎中文文案

#### Scenario: 基础报告展示风险摘要

- **WHEN** 用户打开一个已有风险复核的基础报告
- **THEN** 页面展示每个商机的综合风险等级和风险摘要
- **AND** 页面展示至少一个最需要优先排查的风险触发原因或缓解建议
- **AND** 页面提供进入商机详情查看完整风险复核的入口

#### Scenario: 缺少风险复核时不阻断结果阅读

- **WHEN** 商机详情或基础报告没有可展示的风险复核
- **THEN** 页面仍展示基础商机、需求洞察、货源候选、竞品参考和验证预算内容
- **AND** 页面展示中文说明表示风险复核暂不可用或仍需人工确认
- **AND** 页面不把缺少风险复核误判为任务失败

### Requirement: 风险复核保持谨慎可行动

系统 SHALL 将风险复核表达为待验证风险提示和缓解建议，不得将其包装成正式尽调、合规结论或供应链确认结果。

#### Scenario: 风险复核不宣称风险已经排除

- **WHEN** 系统生成或展示风险复核
- **THEN** 文案不得宣称合规已确认、供应商履约已验证、平台规则无风险、库存风险已排除或等价结论
- **AND** 文案必须表达为初步提示、待验证、需要确认或建议先排查
- **AND** 文案必须包含至少一个用户可以执行的缓解建议

#### Scenario: 风险复核覆盖关键风险类别

- **WHEN** 系统生成风险复核
- **THEN** 每个商机的风险复核至少覆盖产品质量、发货履约、售后和同质化竞争风险
- **AND** 系统在信息可用时覆盖合规、库存积压和平台规则风险
- **AND** 系统不为了填充字段而伪造已经确认的合规、库存或平台规则信息
