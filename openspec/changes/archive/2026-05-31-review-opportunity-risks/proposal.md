## 为什么

当前 MarketPilot 已经能生成基础商机、来源线索、需求洞察、货源候选、竞品参考和验证预算估算，但用户仍缺少“哪些风险会让这个商机不适合做，或必须先验证”的结构化判断。风险复核是连接验证预算和后续行动计划的关键 P1 切片，能让用户在投入样品、内容和首批库存前更谨慎地筛掉高风险方向。

## 变更内容

- 为每个已生成商机补充风险复核，覆盖产品质量、发货履约、售后、合规、库存积压、同质化竞争和平台规则风险。
- 输出每个商机的综合风险等级、风险摘要、分项风险说明、风险触发原因和可行动缓解建议。
- 优先复用现有任务输入、商机结果、来源线索、需求洞察、货源候选、竞品参考和验证预算估算；在信息不足时生成谨慎的待验证风险提示。
- 在研究运行完成验证预算估算后生成风险复核，并保存为可读取的结构化结果。
- 在商机详情和基础报告中展示风险复核摘要、重点风险和非阻塞空状态，避免把初步风险提示包装成已完成的合规或供应链尽调。
- 提供本地和测试环境确定性回退，确保自动化测试不依赖外部 LLM、搜索或网络服务。

## 能力

### 新增能力

- `opportunity-risks`: 定义每个商机的风险复核生成、持久化、读取、前端展示、分项风险、缓解建议和非阻塞失败契约。

### 修改能力

- `opportunity-research`: 研究运行在验证预算估算之后生成风险复核，并记录对应阶段事件；风险复核失败不破坏已完成的基础商机结果。
- `research-progress`: 进度页需要展示风险复核阶段和对应的谨慎中文说明。
- `agent-run-observability`: Agent 阶段历史和 trace 需要覆盖风险复核阶段，便于解释风险复核缺失或生成失败。

## 影响

- 后端新增风险复核领域模型、迁移、模式、仓储、服务和读取 API。
- 数据库新增 `opportunity_risks`，归属关系由 `research_task_id` 和 `opportunity_id` 表达。
- LangGraph 研究流程新增风险复核阶段，并复用已有研究任务、商机、来源、需求洞察、货源候选、竞品参考、验证预算和 Agent 运行事件能力。
- LLM 生成逻辑增加风险复核结构化输出；本地和测试环境增加确定性回退。
- 前端商机详情和基础报告增加风险复核展示区块，并保持缺少风险复核或读取失败时的非阻塞中文空状态。
- 测试覆盖风险复核生成、读取、重新运行替换、运行阶段事件和前端 UI 契约。
