## 为什么

当前 MVP 已经跑通基础商机生成、增强分析、RAG 检索评测和演示就绪检查，但基础商机生成阶段仍直接基于任务输入生成结构化商机，不在生成推荐前做外部前置调研、RAG、竞品深挖或多 Agent 协作。团队需要一个内部生成质量评测切片，用固定中文样例和 rubric 复查已完成研究结果是否完整、守约束、不过度声称，并把结果纳入演示前判断。

## 变更内容

- 新增生成质量评测能力，支持对已完成研究任务运行内部 evaluation run。
- 建立固定中文评测样例和 rubric，覆盖预算、渠道、排除品类、目标人群、供给偏好、结果数量、字段完整性、风险具体性、行动可执行性和谨慎边界。
- 保存评测运行、case 结果、整体状态、中文摘要、指标摘要和安全错误摘要，响应只暴露公开 UUID 与聚合信息。
- 为内部工具或命令提供运行与读取入口，支持对单条已完成任务执行评测，并允许 readiness 读取最近一次生成质量评测摘要。
- 保持评测为内部质量能力，不修改 Agent 生成逻辑、商机排序、公开分享页或用户侧研究结果。

## 能力

### 新增能力

- `generation-quality-evaluation`: 定义生成质量评测集、评测运行、case 结果、rubric 检查、内部读取接口和安全边界。

### 修改能力

- `research-quality-readiness`: 将生成质量评测摘要纳入 readiness 检查项，使演示就绪判断可以体现生成内容语义质量，而不是只做 smoke check。

## 影响

- 后端需要新增生成质量评测的数据模型、迁移、服务、runner 和内部 API 或管理命令。
- readiness runner 需要读取或触发最近一次生成质量评测，并把摘要写入 readiness check 明细。
- 前端公开页面和用户进度页不展示生成质量评测；如需 UI，后续只接入内部工具或我的研究里的轻量内部状态。
- 不新增外部业务流程，不改变研究任务状态机，不删除已有商机、增强分析、RAG 评测、readiness run 或分享快照。
