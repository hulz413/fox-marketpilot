## 为什么

当前 MarketPilot 已经跑通基础商机推荐、来源收集和轻量来源透明度，但每个商机仍主要依赖一段推荐理由，用户难以判断“为什么这个人群真的可能有需求”。下一步需要把需求判断拆成可阅读、可追踪、可验证的洞察，让 P1 质量增强建立在现有来源线索之上。

## 变更内容

- 为每个已生成商机补充需求洞察，说明目标人群、使用场景、购买动机、内容种草点、趋势信号和季节性因素。
- 将需求洞察表达为待验证判断，并尽量关联现有 `demand` 或 `general` 来源线索，不把洞察包装成已证明结论。
- 在研究运行完成基础商机和来源收集后生成需求洞察，并保存为可读取的结构化结果。
- 在商机详情和基础报告中展示需求洞察摘要、来源关联和缺失来源时的谨慎说明。
- 提供本地和测试环境确定性回退，确保自动化测试不依赖外部 LLM 或搜索服务。

## 能力

### 新增能力

- `demand-insights`: 定义每个商机的需求洞察生成、持久化、读取、来源关联和前端展示契约。

### 修改能力

- `opportunity-research`: 研究运行在基础商机结果和来源收集之后生成需求洞察，并记录对应阶段事件；需求洞察失败不破坏已完成的基础商机结果。

## 影响

- 后端新增需求洞察领域模型、迁移、模式、仓储、服务和读取 API。
- LangGraph 研究流程新增需求洞察生成阶段，并复用已有研究任务、商机、来源和 Agent 运行事件能力。
- LLM 生成逻辑增加需求洞察结构化输出；本地和测试环境增加确定性回退。
- 前端商机详情和基础报告增加需求洞察展示区块，并保持缺少来源或洞察时的非阻塞中文空状态。
- 测试覆盖需求洞察生成、读取、来源关联、运行阶段事件和前端 UI 契约。
