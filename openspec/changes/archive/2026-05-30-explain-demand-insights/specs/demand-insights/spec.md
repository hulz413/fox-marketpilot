## ADDED Requirements

### Requirement: 系统持久化商机需求洞察

系统 SHALL 为每个已生成商机保存可读取的结构化需求洞察，并与研究任务、商机和可用来源建立稳定关联。

#### Scenario: 保存需求洞察字段

- **WHEN** 系统成功为商机生成需求洞察
- **THEN** 系统保存需求洞察 UUID
- **AND** 系统将需求洞察关联到对应研究任务和商机
- **AND** 需求洞察包含摘要、目标人群画像、使用场景、购买动机、内容种草点、趋势信号和季节性因素
- **AND** 需求洞察包含来源状态
- **AND** 需求洞察包含创建时间、更新时间和可为空的删除时间
- **AND** 系统不对外暴露需求洞察、研究任务、商机或来源的内部自增 ID

#### Scenario: 关联需求来源

- **WHEN** 需求洞察使用了已收集研究来源
- **THEN** 系统将需求洞察关联到一个或多个未软删除来源
- **AND** 关联来源优先来自同一商机的 `demand` 或 `general` 来源
- **AND** 每个来源关联包含可读的关联说明
- **AND** 来源关联不表示该商机已经被证明成立

#### Scenario: 重新运行替换旧需求洞察

- **WHEN** 一条已完成研究任务被重新运行并成功生成新商机和新需求洞察
- **THEN** 系统将该任务上一轮需求洞察从默认读取中排除
- **AND** 系统保存新运行生成的需求洞察
- **AND** 后续读取该任务需求洞察时默认返回最新未软删除结果

### Requirement: 系统生成需求洞察

系统 SHALL 在基础商机和来源线索可用后，为当前研究任务的每个未软删除商机生成需求洞察。

#### Scenario: 基于商机和来源生成需求洞察

- **WHEN** 基础商机结果已经保存且来源收集阶段已经结束
- **THEN** 系统读取当前任务的未软删除商机
- **AND** 系统读取当前任务的未软删除来源
- **AND** 系统为每个商机生成一条需求洞察
- **AND** 需求洞察说明目标人群、使用场景、购买动机、内容种草点、趋势信号和季节性因素
- **AND** 关键判断尽量关联可用来源

#### Scenario: 缺少需求来源时生成待验证洞察

- **WHEN** 某个商机没有可用的 `demand` 或 `general` 来源
- **THEN** 系统仍可以基于研究任务输入和商机字段生成待验证需求洞察
- **AND** 系统将来源状态标记为缺少来源或 fallback
- **AND** 洞察文案说明该判断需要继续通过公开资料或小批量验证确认

#### Scenario: 需求洞察输出保持谨慎表达

- **WHEN** 系统生成需求洞察摘要、使用场景、趋势信号或来源关联说明
- **THEN** 文案使用“可能”“初步参考”“提示存在相关信号”“待验证”或等价谨慎表达
- **AND** 文案不使用“已证明”“确定有市场”“需求已验证”“趋势已确认”或等价表达

#### Scenario: 本地或测试环境不依赖外部服务

- **WHEN** 系统运行在 local 或 test 环境且未配置 LLM API key
- **THEN** 系统使用确定性 fallback 生成中文需求洞察
- **AND** fallback 结果仍满足需求洞察 schema
- **AND** 自动化测试不依赖外部 LLM、搜索或网络服务

### Requirement: 用户可以读取需求洞察

系统 SHALL 允许前端读取某条研究任务或单个商机的需求洞察，用于商机详情和基础报告展示。

#### Scenario: 读取任务需求洞察列表

- **WHEN** 用户请求一条存在且未软删除研究任务的需求洞察列表
- **THEN** 系统返回该任务未软删除商机关联的未软删除需求洞察
- **AND** 每条需求洞察包含需求洞察 UUID、研究任务 UUID、商机 UUID、摘要、目标人群画像、使用场景、购买动机、内容种草点、趋势信号、季节性因素、来源状态和关联来源摘要
- **AND** 结果按商机推荐排序稳定排列
- **AND** 系统不返回内部自增 ID

#### Scenario: 读取单个商机需求洞察

- **WHEN** 用户请求一个存在且未软删除商机的需求洞察
- **THEN** 系统返回该商机未软删除的需求洞察
- **AND** 返回内容包含关联来源摘要
- **AND** 系统不返回其他商机的需求洞察

#### Scenario: 读取没有需求洞察的任务或商机

- **WHEN** 用户请求的任务或商机尚未生成需求洞察
- **THEN** API 响应保持成功
- **AND** 系统返回空列表或空结果
- **AND** 前端可以展示中文非阻塞空状态

#### Scenario: 读取不存在资源

- **WHEN** 用户请求不存在或已软删除的研究任务或商机
- **THEN** 系统返回未找到错误
- **AND** 系统不返回其他任务或商机的数据

### Requirement: 前端展示需求洞察

前端 SHALL 在商机详情和基础报告中展示需求洞察，帮助用户理解商机为什么可能有需求。

#### Scenario: 商机详情展示完整需求洞察

- **WHEN** 用户打开单个商机详情且该商机存在需求洞察
- **THEN** 页面展示需求洞察摘要、目标人群画像、使用场景、购买动机、内容种草点、趋势信号和季节性因素
- **AND** 页面展示关联来源入口或来源摘要
- **AND** 页面说明这些内容是待验证判断

#### Scenario: 基础报告展示需求洞察摘要

- **WHEN** 用户打开已完成研究任务的基础报告且该任务存在需求洞察
- **THEN** 页面为每个商机展示需求洞察摘要
- **AND** 页面提供进入单个商机详情查看完整洞察的入口
- **AND** 页面不把需求洞察展示为完整市场验证结论

#### Scenario: 需求洞察为空或读取失败

- **WHEN** 需求洞察接口返回空结果或读取失败
- **THEN** 页面展示中文可理解的暂不可用说明
- **AND** 页面保留基础商机推荐、详情或报告主体内容
- **AND** 页面不将缺少需求洞察展示为任务失败

### Requirement: 需求洞察生成可观测且非阻塞

系统 SHALL 为需求洞察生成记录可观测信息，并确保需求洞察失败不破坏已完成的基础商机结果。

#### Scenario: 记录需求洞察生成阶段

- **WHEN** 后台研究运行执行需求洞察生成
- **THEN** 系统记录 `generate_demand_insights` 阶段事件
- **AND** 阶段事件包含任务 UUID、运行 ID、阶段状态、耗时和可安全展示的错误摘要
- **AND** LangSmith tracing 启用时，该阶段与同一研究运行 trace 关联

#### Scenario: 需求洞察生成失败

- **WHEN** 基础商机结果已经保存但需求洞察生成失败
- **THEN** 研究任务仍可保持 `completed`
- **AND** 系统不删除已生成的商机结果或来源
- **AND** 系统记录需求洞察失败的阶段事件和应用日志
- **AND** 需求洞察读取接口返回空结果
- **AND** 用户可见错误摘要不包含敏感凭证、原始异常堆栈或内部自增 ID
