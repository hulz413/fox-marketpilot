## ADDED Requirements

### Requirement: 研究任务列表提供进度入口

系统 SHALL 让研究任务列表可以进入单个任务的进度页，使用户从列表继续查看运行状态、失败原因和下一步操作。

#### Scenario: 运行中任务进入进度页

- **WHEN** 用户在研究任务列表查看状态为 `queued` 或 `running` 的任务
- **THEN** 前端展示进入任务进度页的入口
- **AND** 该入口使用研究任务 UUID
- **AND** 前端不要求用户先进入商机列表或报告页

#### Scenario: 失败任务进入进度页

- **WHEN** 用户在研究任务列表查看状态为 `failed` 的任务
- **THEN** 前端展示进入任务进度页的入口
- **AND** 进度页可以展示该任务的失败原因和重新运行入口

#### Scenario: 已完成任务保留结果入口

- **WHEN** 用户在研究任务列表查看状态为 `completed` 的任务
- **THEN** 前端仍提供商机推荐入口
- **AND** 前端仍提供基础报告入口
- **AND** 前端可以提供进度页入口用于查看本次运行阶段历史

### Requirement: 新建研究成功后进入任务进度页

前端 SHALL 在用户创建并启动研究任务后进入新任务的进度页，而不是只返回任务列表。

#### Scenario: 创建并启动后跳转进度页

- **WHEN** 用户在新建研究页面提交有效需求并成功启动研究运行
- **THEN** 前端跳转到该研究任务的进度页
- **AND** 进度页展示该任务的运行中状态或最新阶段
- **AND** 用户无需从任务列表再次寻找刚创建的任务

#### Scenario: 启动失败后保留错误信息

- **WHEN** 用户创建任务成功但启动研究运行失败
- **THEN** 前端展示中文可理解的失败信息
- **AND** 用户可以进入任务进度页查看任务状态或重新运行

### Requirement: 任务进度页可以启动或重新运行研究任务

系统 SHALL 允许用户从研究任务进度页对可运行任务启动或重新运行基础商机研究。

#### Scenario: 从已创建任务启动研究

- **WHEN** 用户在状态为 `created` 的任务进度页点击启动研究
- **THEN** 系统为该任务生成 run ID
- **AND** 系统将任务状态更新为 `queued`
- **AND** 进度页展示新的运行状态

#### Scenario: 从失败任务重新运行研究

- **WHEN** 用户在状态为 `failed` 的任务进度页点击重新运行
- **THEN** 系统为该任务生成新的 run ID
- **AND** 系统清空上一轮失败原因
- **AND** 系统将任务状态更新为 `queued`
- **AND** 进度页展示新 run 的进度

#### Scenario: 避免重复启动运行中任务

- **WHEN** 用户在状态为 `queued` 或 `running` 的任务进度页尝试启动研究
- **THEN** 系统不创建重复运行
- **AND** 前端继续展示当前 run 的进度
