## ADDED Requirements

### Requirement: 系统持久化货源候选

系统 SHALL 为每个已生成商机保存可读取的结构化货源候选，并与研究任务、商机和可用来源建立稳定关联。

#### Scenario: 保存货源候选基础字段

- **WHEN** 系统成功为商机生成货源候选
- **THEN** 系统保存货源候选 UUID
- **AND** 系统将货源候选关联到对应研究任务和商机
- **AND** 每个货源候选包含排序、候选名称、可能供给市场、搜索关键词、价格区间、起订量、规格/材质/款式参考、供应商确认问题、推荐说明和来源状态
- **AND** 货源候选包含创建时间、更新时间和可为空的删除时间
- **AND** 系统不对外暴露货源候选、研究任务、商机或来源的内部自增 ID

#### Scenario: 每个商机保存多个候选

- **WHEN** 系统为一条研究任务生成货源候选
- **THEN** 每个未软删除商机至少保存 2 个货源候选
- **AND** 同一商机下的货源候选排序不重复
- **AND** 排序可用于商机详情和报告摘要展示

#### Scenario: 关联货源来源

- **WHEN** 货源候选使用了已收集研究来源
- **THEN** 系统将货源候选关联到一个或多个未软删除来源
- **AND** 关联来源优先来自同一商机的 `supply` 或 `general` 来源
- **AND** 每个来源关联包含可读的关联说明
- **AND** 来源关联不表示供应商、库存、价格或履约能力已经被确认

#### Scenario: 重新运行替换旧货源候选

- **WHEN** 一条已完成研究任务被重新运行并成功生成新商机和新货源候选
- **THEN** 系统将该任务上一轮货源候选从默认读取中排除
- **AND** 系统保存新运行生成的货源候选
- **AND** 后续读取该任务货源候选时默认返回最新未软删除结果

### Requirement: 系统生成货源候选

系统 SHALL 在基础商机、来源线索和需求洞察阶段结束后，为当前研究任务的每个未软删除商机生成货源候选。

#### Scenario: 基于商机和来源生成货源候选

- **WHEN** 基础商机结果已经保存且来源收集阶段已经结束
- **THEN** 系统读取当前任务的未软删除商机
- **AND** 系统读取当前任务的未软删除来源
- **AND** 系统为每个商机生成至少 2 个货源候选
- **AND** 货源候选说明可能供给市场、搜索关键词、价格区间、起订量、规格/材质/款式参考和供应商确认问题
- **AND** 关键判断尽量关联可用来源

#### Scenario: 缺少货源来源时生成待验证候选

- **WHEN** 某个商机没有可用的 `supply` 或 `general` 来源
- **THEN** 系统仍可以基于研究任务输入、供给来源偏好和商机字段生成待验证货源候选
- **AND** 系统将来源状态标记为缺少来源或 fallback
- **AND** 候选文案说明价格、起订量、库存和履约能力需要继续向供应商确认

#### Scenario: 货源候选输出保持谨慎表达

- **WHEN** 系统生成候选名称、推荐说明、规格参考、确认问题或来源关联说明
- **THEN** 文案使用“候选”“初步参考”“可能”“需要确认”“待验证”或等价谨慎表达
- **AND** 文案不使用“已确认供给”“供应商已核验”“价格已确认”“库存已确认”或等价表达

#### Scenario: 本地或测试环境不依赖外部服务

- **WHEN** 系统运行在 local 或 test 环境且未配置 LLM API key
- **THEN** 系统使用确定性 fallback 生成中文货源候选
- **AND** fallback 结果仍满足货源候选 schema
- **AND** 自动化测试不依赖外部 LLM、搜索或网络服务

### Requirement: 用户可以读取货源候选

系统 SHALL 允许前端读取某条研究任务或单个商机的货源候选，用于商机详情和基础报告展示。

#### Scenario: 读取任务货源候选列表

- **WHEN** 用户请求一条存在且未软删除研究任务的货源候选列表
- **THEN** 系统返回该任务未软删除商机关联的未软删除货源候选
- **AND** 每条货源候选包含货源候选 UUID、研究任务 UUID、商机 UUID、排序、候选名称、可能供给市场、搜索关键词、价格区间、起订量、规格/材质/款式参考、供应商确认问题、推荐说明、来源状态和关联来源摘要
- **AND** 结果按商机推荐排序和候选排序稳定排列
- **AND** 系统不返回内部自增 ID

#### Scenario: 读取单个商机货源候选

- **WHEN** 用户请求一个存在且未软删除商机的货源候选列表
- **THEN** 系统返回该商机未软删除的货源候选
- **AND** 返回内容包含关联来源摘要
- **AND** 系统不返回其他商机的货源候选

#### Scenario: 读取没有货源候选的任务或商机

- **WHEN** 用户请求的任务或商机尚未生成货源候选
- **THEN** API 响应保持成功
- **AND** 系统返回空列表
- **AND** 前端可以展示中文非阻塞空状态

#### Scenario: 读取不存在资源

- **WHEN** 用户请求不存在或已软删除的研究任务或商机
- **THEN** 系统返回未找到错误
- **AND** 系统不返回其他任务或商机的数据

### Requirement: 前端展示货源候选

前端 SHALL 在商机详情和基础报告中展示货源候选，帮助用户理解商机可以如何继续找货和询盘。

#### Scenario: 商机详情展示完整货源候选

- **WHEN** 用户打开单个商机详情且该商机存在货源候选
- **THEN** 页面展示候选名称、可能供给市场、搜索关键词、价格区间、起订量、规格/材质/款式参考、供应商确认问题和推荐说明
- **AND** 页面展示关联来源入口或来源摘要
- **AND** 页面说明候选货源仍需向供应商确认

#### Scenario: 基础报告展示货源候选摘要

- **WHEN** 用户打开已完成研究任务的基础报告且该任务存在货源候选
- **THEN** 页面为每个商机展示货源候选摘要
- **AND** 页面提供进入单个商机详情查看完整候选的入口
- **AND** 页面不把货源候选展示为已确认供应商或已核验商品

#### Scenario: 货源候选为空或读取失败

- **WHEN** 货源候选接口返回空结果或读取失败
- **THEN** 页面展示中文可理解的暂不可用说明
- **AND** 页面保留基础商机推荐、详情或报告主体内容
- **AND** 页面不将缺少货源候选展示为任务失败

### Requirement: 货源候选生成可观测且非阻塞

系统 SHALL 为货源候选生成记录可观测信息，并确保货源候选失败不破坏已完成的基础商机结果。

#### Scenario: 记录货源候选生成阶段

- **WHEN** 后台研究运行执行货源候选生成
- **THEN** 系统记录 `generate_supply_candidates` 阶段事件
- **AND** 阶段事件包含任务 UUID、运行 ID、阶段状态、耗时和可安全展示的错误摘要
- **AND** LangSmith tracing 启用时，该阶段与同一研究运行 trace 关联

#### Scenario: 货源候选生成失败

- **WHEN** 基础商机结果已经保存但货源候选生成失败
- **THEN** 研究任务仍可保持 `completed`
- **AND** 系统不删除已生成的商机结果、来源或需求洞察
- **AND** 系统记录货源候选失败的阶段事件和应用日志
- **AND** 货源候选读取接口返回空结果
- **AND** 用户可见错误摘要不包含敏感凭证、原始异常堆栈或内部自增 ID
