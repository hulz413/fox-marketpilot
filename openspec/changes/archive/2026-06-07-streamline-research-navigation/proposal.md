## 为什么

当前一级导航同时暴露研究任务、研究历史、商机推荐和最终报告，但这些入口都围绕同一条研究任务上下文展开，用户会在“历史点报告、任务点结果”的不同路径里迷失。

本 change 将研究相关入口收束为更清晰的“开始研究 / 找到研究 / 阅读结果”流转，让完成任务后的商机、报告、来源和行动计划都从同一个研究上下文进入。

## 变更内容

- 将全局导航中的任务与历史入口合并为一个研究记录/工作台型入口，避免两个相似列表重复展示同一批研究任务。
- 将全局商机推荐和最终报告入口降级为有上下文的结果入口，缺少任务上下文时引导用户从统一研究列表选择任务。
- 统一完成任务后的主操作文案和目标，让用户从列表进入同一条研究的结果视图，再在任务内切换商机推荐、报告、来源等内容。
- 在统一研究列表中用状态筛选或分组表达运行中、已完成、失败和全部记录，保留重新运行、查看进度、查看结果和演示状态信息。
- 保留现有研究任务、商机结果、报告分享和质量就绪检查数据模型，不新增后端存储模型。
- 不修改 Agent 生成逻辑、商机排序算法、来源收集策略、LangSmith tracing 或报告分享公开页能力。

## 能力

### 新增能力

- 无

### 修改能力

- `product-skeleton`: 调整全局导航、缺少任务上下文的结果入口和任务内导航的用户体验要求。
- `research-tasks`: 将任务列表与历史列表的用户可见入口收束为统一研究列表，并明确状态筛选、主操作和重新运行入口。
- `opportunity-results`: 调整商机推荐和基础报告的进入方式，使其作为单条研究上下文内的结果视图，而不是并列的无上下文一级目的地。

## 影响

- 前端应用壳、全局导航、面包屑、研究任务页、研究历史页、商机推荐入口页、报告入口页、任务上下文导航和相关 i18n 文案。
- 前端列表 UI 需要复用现有研究任务 API，并保留小屏卡片和桌面表格体验。
- 现有后端 API、数据库迁移、Agent 编排、Celery worker、LangSmith tracing、来源收集和报告分享公开页不在本 change 范围内。
