## 为什么

当前 MVP 已经能生成商机推荐、来源线索、需求洞察、货源候选、竞品参考、验证预算和风险复核，但最终结果仍偏“判断材料”，用户还需要自己把这些信息整理成可执行的验证步骤。

本 change 补齐“下一步怎么开始验证”的最后一公里，让每个商机都有可直接用于小预算试跑的行动计划。

## 变更内容

- 为每个商机生成首批验证计划，覆盖验证目标、首批验证方式、内容种草角度、商品标题/卖点建议、供应商询盘话术和上架前检查清单。
- 在基础商机研究运行中，于风险复核之后追加行动计划生成阶段；该阶段失败不覆盖已生成的基础商机结果和既有增强信息。
- 提供按任务和按商机读取行动计划的 API，外部响应使用任务/商机 UUID，不暴露内部自增 ID。
- 在商机详情和基础报告中展示行动计划，帮助用户从报告直接进入可执行验证动作。
- 在研究进度页和 Agent 可观测事件中展示行动计划阶段、状态、耗时和失败摘要。
- 非目标：不自动下单、不自动联系供应商、不自动发布内容、不接入第三方平台账号、不把行动计划表达为保证成交或保证回本的承诺。

## 能力

### 新增能力

- `action-plans`: 定义商机行动计划的生成、持久化、读取、前端展示和非阻塞可观测契约，帮助用户把商机判断转化为小预算验证步骤。

### 修改能力

- `opportunity-research`: 将行动计划生成接入现有 LangGraph 研究运行，位于风险复核之后，并保持失败不阻断基础结果。
- `research-progress`: 在进度页阶段时间线、阶段文案和失败但完成的展示中加入行动计划阶段。
- `agent-run-observability`: 记录行动计划生成阶段的 trace、阶段事件、耗时、结果状态和安全失败摘要。

## 影响

- 后端：新增行动计划模块、数据模型、迁移、schema、repository、service 和 API routes；扩展研究运行阶段枚举和 LangGraph 节点。
- 前端：新增行动计划 API 类型与读取函数；在商机详情、基础报告和研究进度页展示行动计划内容与阶段状态。
- 测试：新增行动计划后端单元/集成测试，扩展研究运行、进度和前端契约测试。
- 外部系统：复用现有 OpenAI-compatible LLM provider 和 LangSmith tracing；不新增第三方平台集成或账号权限。
