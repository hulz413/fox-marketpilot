## ADDED Requirements

### Requirement: 产品骨架采用企业列表型应用壳

MarketPilot 前端 SHALL 提供浅色企业列表型应用壳，用于展示商机顾问 Agent 的产品骨架。

#### Scenario: 首次打开产品

- **WHEN** 用户打开前端首页
- **THEN** 页面展示 MarketPilot 产品界面，而不是项目骨架技术占位页
- **AND** 页面包含左侧边栏、顶部页面栏和中间主内容区
- **AND** 页面视觉为浅色企业后台风格
- **AND** 页面使用 `#2F6F4E` 深翡翠绿作为产品主色
- **AND** 页面不使用暗色主题作为默认界面

#### Scenario: 查看产品定位

- **WHEN** 用户查看应用壳
- **THEN** 用户能看到产品名称 MarketPilot
- **AND** 用户能看到商机顾问 Agent 相关中文说明
- **AND** 页面文案默认使用中文

### Requirement: 左侧导航只包含 P0 必要入口

产品骨架 SHALL 在左侧边栏提供 P0 演示闭环所需的一级入口，并避免暴露过多 P1 分析模块。

#### Scenario: 查看左侧导航

- **WHEN** 用户查看左侧边栏
- **THEN** 导航包含新建研究、研究任务、商机推荐、最终报告和研究历史入口
- **AND** 每个入口都有可理解的中文名称
- **AND** 每个入口有明确的可点击目标

#### Scenario: 避免一级入口过载

- **WHEN** 用户查看左侧边栏
- **THEN** 导航不把来源依据、供给候选、竞品情报、风险复核和验证计划作为一级入口展示
- **AND** 这些后续分析能力只作为详情页或报告页内部区块预留

### Requirement: 产品骨架提供关键页面路由

产品骨架 SHALL 提供可访问的页面骨架，使用户能浏览 P0 研究闭环的主要阶段。

#### Scenario: 访问新建研究入口

- **WHEN** 用户点击新建研究入口或顶部新建研究按钮
- **THEN** 用户进入新建研究页面或新建研究区域
- **AND** 页面展示自然语言研究需求输入骨架
- **AND** 页面展示预算、目标渠道、偏好或限制条件等关键输入占位

#### Scenario: 访问研究任务列表

- **WHEN** 用户点击研究任务入口
- **THEN** 用户看到研究任务列表骨架
- **AND** 列表至少展示任务标题、状态、当前阶段、创建时间和操作入口
- **AND** 列表适配企业列表型表格风格

#### Scenario: 访问商机推荐

- **WHEN** 用户点击商机推荐入口
- **THEN** 用户看到商机推荐列表骨架
- **AND** 列表至少展示商机名称、目标人群、验证预算、风险等级、推荐优先级和详情入口
- **AND** 页面能表达每次研究应输出 3-5 个商机推荐

#### Scenario: 访问商机详情

- **WHEN** 用户从商机推荐列表打开单个商机
- **THEN** 用户看到商机详情骨架
- **AND** 详情至少包含商机名称、目标人群、推荐理由、适合渠道、价格带、利润空间、风险等级和下一步建议占位

#### Scenario: 访问最终报告

- **WHEN** 用户点击最终报告入口或从任务/商机进入报告
- **THEN** 用户看到最终报告页面骨架
- **AND** 报告页面能独立展示推荐排序、商机摘要、风险摘要和下一步行动摘要占位

#### Scenario: 访问研究历史

- **WHEN** 用户点击研究历史入口
- **THEN** 用户看到研究历史列表骨架
- **AND** 列表至少展示历史任务标题、创建时间、状态、报告入口和重新运行入口占位

### Requirement: 产品骨架覆盖关键页面状态

产品骨架 SHALL 展示 P0 演示闭环需要的关键页面状态，便于后续真实流程接入和验收。

#### Scenario: 首次空状态

- **WHEN** 用户没有任何研究任务
- **THEN** 页面展示清晰空状态
- **AND** 空状态提供新建研究入口
- **AND** 空状态提供 2-3 个中文示例研究需求

#### Scenario: 运行中状态

- **WHEN** 用户查看运行中的研究任务
- **THEN** 页面展示任务运行中状态
- **AND** 页面展示当前阶段占位，例如收集来源、分析供给、生成推荐或生成报告
- **AND** 页面预留后续 SSE 状态更新的显示区域

#### Scenario: 失败状态

- **WHEN** 用户查看失败的研究任务
- **THEN** 页面展示可理解的失败原因占位
- **AND** 页面提供重新运行入口占位

#### Scenario: 完成状态

- **WHEN** 用户查看已完成研究任务
- **THEN** 页面展示进入商机推荐和最终报告的入口
- **AND** 用户能从完成任务继续浏览推荐列表、商机详情和报告骨架

### Requirement: 产品骨架保持实现边界

产品骨架 SHALL 只建立前端页面结构和占位状态，不实现真实业务执行。

#### Scenario: 验证业务边界

- **WHEN** 开发者检查本 change 的实现
- **THEN** 前端不要求真实后端 API 才能浏览产品骨架
- **AND** 前端不创建真实研究任务
- **AND** 前端不执行真实 Agent 流程
- **AND** 前端不持久化真实业务数据

#### Scenario: 验证技术栈边界

- **WHEN** 开发者检查依赖和组件
- **THEN** 产品骨架使用既有 Next.js、TypeScript、Tailwind CSS、shadcn/ui 和 lucide icon 能力
- **AND** 本 change 不新增第三方运行时依赖
