## 为什么

当前 MarketPilot 已经能生成基础商机、收集公开来源、解释需求洞察、给出货源候选并展示竞品参考，但用户还缺少“这个商机是否值得用小预算验证、第一批大概要准备多少钱、毛利空间是否粗略成立”的判断。利润与验证预算粗算是连接货源、竞品和后续风险/行动计划的关键 P1 切片，适合作为下一步补强。

## 变更内容

- 为每个已生成商机补充验证预算估算，覆盖预估拿货成本、预估售价、粗略毛利空间、首批验证数量、首批验证预算和关键验证假设。
- 优先复用现有商机价格带、粗略利润空间、任务预算、期望利润、货源候选和竞品参考；在信息不足时生成谨慎的待验证粗算。
- 在研究运行完成来源收集、需求洞察、货源候选和竞品参考后生成验证预算估算，并保存为可读取的结构化结果。
- 在商机详情和基础报告中展示预算摘要、关键假设、缺少数据时的非阻塞说明，以及与小预算验证相关的谨慎文案。
- 提供本地和测试环境确定性回退，确保自动化测试不依赖外部 LLM、搜索或网络服务。

## 能力

### 新增能力

- `validation-budgets`: 定义每个商机的验证预算估算生成、持久化、读取、前端展示、关键假设和非阻塞失败契约。

### 修改能力

- `opportunity-research`: 研究运行在竞品参考之后生成验证预算估算，并记录对应阶段事件；验证预算失败不破坏已完成的基础商机结果。
- `research-progress`: 进度页需要展示验证预算估算阶段和对应的谨慎中文说明。
- `agent-run-observability`: Agent 阶段历史和 trace 需要覆盖验证预算估算阶段，便于解释预算估算缺失或生成失败。

## 影响

- 后端新增验证预算领域模型、迁移、模式、仓储、服务和读取 API。
- 数据库新增 `validation_budgets`，归属关系由 `research_task_id` 和 `opportunity_id` 表达。
- LangGraph 研究流程新增验证预算估算阶段，并复用已有研究任务、商机、来源、需求洞察、货源候选、竞品参考和 Agent 运行事件能力。
- LLM 生成逻辑增加验证预算结构化输出；本地和测试环境增加确定性回退。
- 前端商机详情和基础报告增加验证预算展示区块，并保持缺少预算估算或读取失败时的非阻塞中文空状态。
- 测试覆盖验证预算生成、读取、重新运行替换、运行阶段事件和前端 UI 契约。
