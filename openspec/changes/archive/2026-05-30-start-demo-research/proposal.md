## Why

当前核心研究闭环已经可以从手写需求创建真实任务并进入进度页，但首次演示仍要求用户理解表单并手动提交。`start-demo-research` 用一键中文示例启动真实研究任务，降低演示开场成本，并收齐 roadmap 中最后一个 P0 切片。

## What Changes

- 在首次访问或无任务空状态中提供 2-3 个可直接启动的中文示例研究入口，而不是只展示静态示例文本。
- 在新建研究页面保留示例填入能力，并提供能用示例一键创建任务并启动研究运行的主流程。
- 示例启动必须复用真实研究任务创建和运行接口，创建后进入新任务进度页，后续继续使用已有商机列表、详情、报告、来源和需求洞察闭环。
- 示例启动失败时展示中文错误，并避免把静态 demo 结果伪装成真实研究结果。

## Capabilities

### New Capabilities

- 无。

### Modified Capabilities

- `product-skeleton`: 首次空状态和产品入口从静态中文示例升级为可行动的一键演示入口。
- `research-tasks`: 新建研究与任务空状态支持从预置中文示例创建并启动真实研究任务。

## Impact

- 前端：产品空状态、新建研究页面、示例需求数据结构、任务列表空状态和相关 UI 状态。
- API：复用已有研究任务创建 API 和启动运行 API，不要求新增后端端点。
- Agent/Worker：复用已有基础商机研究运行链路，不改变 LangGraph 节点、Celery 投递或商机生成逻辑。
- 数据：新增任务仍保存为普通真实研究任务，不引入静态 demo 结果或特殊持久化模型。
- 依赖：不新增第三方运行时依赖。
