## ADDED Requirements

### Requirement: 进度页展示 RAG 证据索引阶段

系统 SHALL 在研究任务进度页和阶段时间线中展示 RAG 证据索引阶段，让用户理解系统正在把已收集来源整理为后续分析可用的检索证据。

#### Scenario: RAG 索引阶段展示谨慎文案

- **WHEN** 用户打开当前阶段为 `index_rag_evidence` 的研究任务进度页
- **THEN** 系统展示正在整理公开来源证据的中文说明
- **AND** 文案表达为公开线索整理、初步参考或待验证证据
- **AND** 文案不宣称系统已经完成市场核验、竞品核验或售价确认

#### Scenario: RAG 索引完成后展示时间线

- **WHEN** 用户打开已完成研究任务的进度页且当前 run 存在 `index_rag_evidence` 阶段事件
- **THEN** 阶段时间线展示 RAG 证据索引阶段的完成状态、耗时或完成时间
- **AND** 系统仍展示进入商机推荐和基础报告的入口
- **AND** 系统不要求用户理解 embedding、pgvector 或内部检索实现细节

#### Scenario: RAG 索引失败但任务完成

- **WHEN** 用户打开一条商机结果已完成但 RAG 证据索引失败或被跳过的研究任务进度页
- **THEN** 系统仍展示任务为已完成或可查看结果的状态
- **AND** 系统可以在阶段时间线中展示证据索引暂不可用、已跳过或部分失败的中文说明
- **AND** 系统不将 RAG 索引失败误判为基础商机生成失败
