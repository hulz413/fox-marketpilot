## ADDED Requirements

### Requirement: 系统创建最终报告分享链接

系统 SHALL 允许用户为已完成且已有商机结果的研究任务创建最终报告分享链接。

#### Scenario: 为已完成任务创建分享

- **WHEN** 用户请求分享一条存在、未软删除且已有当前商机结果的研究任务
- **THEN** 系统创建一条报告分享记录
- **AND** 系统生成不可猜测的分享 token
- **AND** 系统保存创建分享时的报告快照
- **AND** 系统返回可在线浏览的分享链接或分享 token

#### Scenario: 没有商机结果时拒绝分享

- **WHEN** 用户请求分享一条尚未生成商机结果的研究任务
- **THEN** 系统拒绝创建分享记录
- **AND** 系统返回中文可理解错误
- **AND** 系统不创建静态 demo 分享页

#### Scenario: 同一任务可以创建多个分享快照

- **WHEN** 用户多次请求分享同一条可分享研究任务
- **THEN** 系统可以创建多条分享记录
- **AND** 每条分享记录拥有不同分享 token
- **AND** 每条分享记录保存各自创建时的报告快照

### Requirement: 系统保存分享报告快照

系统 SHALL 将分享报告保存为结构化快照，确保分享页内容稳定且不暴露内部实现细节。

#### Scenario: 保存公开报告快照字段

- **WHEN** 系统创建报告分享
- **THEN** 快照包含任务需求摘要、商机推荐排序、商机详情摘要、需求洞察、货源候选、竞品参考、验证预算、风险复核、行动计划和来源线索
- **AND** 快照包含创建分享时间和中文谨慎边界说明
- **AND** 快照不要求对象存储文件或导出 HTML 文件才能在线浏览

#### Scenario: 分享快照不暴露内部字段

- **WHEN** 系统保存或返回分享报告快照
- **THEN** 快照不得包含研究任务、商机、来源或分享记录的内部自增 ID
- **AND** 快照不得包含 run ID、trace ID、trace URL、阶段事件、错误堆栈、对象存储配置、LLM/provider 凭证或内部日志
- **AND** 公开 API 响应只返回报告阅读所需字段

#### Scenario: 增强分析缺失时仍可分享

- **WHEN** 研究任务已有商机结果但缺少来源、需求洞察、货源候选、竞品参考、验证预算、风险复核或行动计划中的部分增强数据
- **THEN** 系统仍可以创建分享报告快照
- **AND** 缺失区块以中文说明表示暂不可用或仍需人工确认
- **AND** 系统不把增强数据缺失误判为分享失败

### Requirement: 用户可以管理报告分享记录

系统 SHALL 提供创建分享、查看任务分享记录和撤销分享的 API。

#### Scenario: 查看任务分享列表

- **WHEN** 用户请求一条存在且未软删除研究任务的分享记录列表
- **THEN** 系统返回该任务未软删除的分享记录
- **AND** 结果按创建时间倒序或其他稳定顺序展示
- **AND** 响应不暴露内部自增 ID

#### Scenario: 撤销分享链接

- **WHEN** 用户请求撤销一条存在且未软删除的分享记录
- **THEN** 系统将该分享标记为 `revoked`
- **AND** 系统记录撤销时间
- **AND** 后续通过该分享 token 公开读取报告时不可访问

#### Scenario: 报告页一次撤销当前任务所有可访问分享

- **WHEN** 用户在最终报告页请求撤销分享
- **THEN** 系统撤销该任务当前所有 `active` 且未软删除的分享记录
- **AND** 撤销完成后报告页不再展示可访问分享链接
- **AND** 用户不需要重复点击撤销旧的 active 分享链接

#### Scenario: 重新运行后旧分享保持不变

- **WHEN** 一条已分享报告的研究任务被重新运行并生成新的商机结果
- **THEN** 旧分享记录和报告快照保持原内容不变
- **AND** 用户再次分享时系统创建新的分享记录和新的分享 token
- **AND** 系统不使用旧分享覆盖新报告，也不使用新报告覆盖旧分享

### Requirement: 公开分享页可以在线浏览报告

系统 SHALL 提供可通过分享 token 在线浏览的只读报告页。

#### Scenario: 打开有效分享链接

- **WHEN** 访问者打开有效且未撤销的分享链接
- **THEN** 页面展示在线报告
- **AND** 报告展示任务摘要、推荐排序、商机详情、来源线索和谨慎边界说明
- **AND** 页面不要求访问者下载 HTML 文件、登录账号或访问后台任务页

#### Scenario: 分享页采用阅读型布局

- **WHEN** 访问者在桌面浏览器打开分享报告
- **THEN** 页面展示清晰标题、摘要指标、目录或导航、商机分析区块和来源区块
- **AND** 关键结论、风险、预算和行动建议易于扫描
- **AND** 页面不要求横向滚动宽表格才能理解主要内容

#### Scenario: 手机浏览分享报告

- **WHEN** 访问者在手机或窄屏浏览器打开分享报告
- **THEN** 页面使用单列或卡片式布局展示核心内容
- **AND** 目录、商机区块、来源列表和行动计划不会互相遮挡
- **AND** 文本、按钮和链接在窄屏下保持可读和可点击

#### Scenario: 分享不存在或已撤销

- **WHEN** 访问者打开不存在、已软删除或已撤销的分享 token
- **THEN** 页面展示中文不可访问或已撤销说明
- **AND** 系统不返回报告快照内容
- **AND** 系统不暴露该 token 是否关联内部任务 ID

### Requirement: 分享报告保持只读和谨慎边界

系统 SHALL 将分享报告表达为只读参考，不得暴露可修改任务或自动执行经营动作的能力。

#### Scenario: 分享页不展示内部操作

- **WHEN** 访问者打开分享报告
- **THEN** 页面不展示重新运行、编辑任务、查看进度、LangSmith trace、后台调试或删除任务入口
- **AND** 页面不允许访问者修改任务、商机、来源或行动计划
- **AND** 页面只展示报告阅读和外部来源访问所需操作

#### Scenario: 分享报告不承诺经营结果

- **WHEN** 系统生成或展示分享报告
- **THEN** 报告不得宣称保证成交、保证回本、供应商已确认、平台审核必过或市场已经完整核验
- **AND** 报告必须表达为建议、初步参考、待验证或需要人工确认
- **AND** 报告必须说明系统不会自动下单、自动联系供应商或自动发布内容

#### Scenario: 分享页外部链接安全打开

- **WHEN** 分享报告展示来源链接
- **THEN** 外部链接在新标签打开
- **AND** 外部链接使用 `rel="noreferrer"` 或等价安全属性
- **AND** 页面不把来源链接伪装为系统已核验结论
