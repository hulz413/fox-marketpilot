## 背景

MarketPilot 现在已经能在任务报告页汇总商机推荐、需求洞察、货源候选、竞品参考、验证预算、风险复核、行动计划和来源线索。这个报告可以在产品内阅读，但还不能一键生成可分享的在线链接。

用户最新目标不是“导出一个文件”，而是“最终生成的报告可以分享，别人直接在线浏览”。因此本 change 从文件导出转向在线分享：后端保存一份报告快照，前端提供公开只读分享页。

## 目标 / 非目标

**目标：**

- 为已完成且已有商机结果的研究任务创建可分享报告链接。
- 分享页在线浏览，不要求下载 HTML 文件。
- 分享内容是创建分享时的报告快照，后续重新运行任务不会改变旧分享内容。
- 分享页布局合理，适配桌面和手机，适合直接发给他人阅读。
- 支持复制分享链接、打开分享页、查看已有分享记录和撤销分享。
- 分享失败不影响原报告页继续阅读。
- 自动化测试不依赖对象存储、外部网络或真实浏览器服务。

**非目标：**

- 不导出 HTML、PDF、Markdown、DOCX 或其他文件。
- 不使用对象存储保存报告文件。
- 不做公开分享广场、SEO 索引、访问密码、团队权限、登录态分享或邮件发送。
- 不允许分享页重新触发研究、重新生成分析或修改任务数据。
- 不把分享动作接入 LangGraph 主研究流程。

## 技术决策

### 1. 新增 `report_shares` 独立模型

分享记录作为报告交付产物保存，不直接扩展 `research_tasks` 表。建议新增 `report_shares` 表：

- `id`: 内部自增主键。
- `uuid`: 对外公开 UUID。
- `research_task_id`: 内部外键，关联研究任务。
- `share_token`: URL-safe 随机 token，用于公开分享链接，不使用任务 UUID。
- `title`: 分享报告标题快照。
- `snapshot`: JSON，保存公开分享页需要展示的报告数据。
- `status`: `active` 或 `revoked`。
- `revoked_at`: 分享被撤销的时间，可为空。
- `created_at`、`updated_at`、`deleted_at`。

该表遵循项目数据库通用字段规范。默认任务内分享列表只返回 `deleted_at` 为空的记录；公开读取只允许 `status = active` 且 `deleted_at IS NULL` 的分享。API 响应使用分享 UUID、分享 token 和研究任务 UUID，不暴露内部自增 ID。

替代方案：直接在公开接口里通过任务 UUID 实时读取报告。这更简单，但任务重新运行后旧分享内容会变化，也更容易暴露任务结构和调试字段。分享快照更符合“最终生成的报告可交付”的直觉。

### 2. 分享 token 使用不可猜测随机值

公开 URL 不使用任务 UUID 或导出 UUID 作为唯一访问凭证。服务端生成高熵 URL-safe token，例如 128-bit 以上随机值编码为字符串。前端分享路径建议为：

```text
/share/reports/{share_token}
```

后端公开读取接口建议为：

```text
GET /api/v1/report-shares/{share_token}
```

这样链接可以直接在线浏览，同时降低通过枚举任务 UUID 访问报告的风险。第一版不做密码和权限；后续如要强化访问控制，可以在同一模型上增加过期时间、访问密码或 owner。

### 3. 分享快照保存结构化 JSON

第一版不生成 HTML 文件，也不使用对象存储。报告快照保存为结构化 JSON，包含公开分享页需要的字段：

- 任务标题、需求、预算、渠道、创建分享时间。
- 推荐排序和商机基础字段。
- 每个商机的需求洞察、货源候选、竞品参考、验证预算、风险复核和行动计划摘要。
- 来源线索标题、链接、摘要、类型和支撑强度。
- 谨慎边界文案，例如不自动下单、不自动联系供应商、不自动发布内容，利润和风险均需人工确认。

快照生成时需要主动排除内部字段：内部自增 ID、run ID、trace ID、trace URL、阶段事件、错误堆栈、对象存储配置、LLM/provider 配置和完整内部日志。

替代方案：保存完整 HTML 字符串。这样公开读取时渲染更直接，但会重新引入 XSS 和文件式交付问题；结构化 JSON 更容易由前端复用现有报告组件和安全渲染。

### 4. 分享页由前端在线渲染

新增公开分享报告页面，例如 `frontend/src/app/share/reports/[token]/page.tsx`。该页面不使用完整企业应用壳作为默认布局，可以使用更轻的阅读型布局：报告标题、摘要指标、目录、推荐排序、商机区块、来源区块和边界说明。

页面要求：

- 桌面端信息密度高但不拥挤，便于扫描和比较。
- 手机端采用单列或卡片式布局，不出现页面级横向溢出。
- 分享页只读，不展示重新运行、编辑、内部任务进度、LangSmith 或调试入口。
- 来源链接可打开到外部站点，使用 `target="_blank"` 和 `rel="noreferrer"`。
- 缺失增强数据时展示中文非阻塞说明。

可以支持轻量客户端交互，例如目录跳转、商机折叠、来源类型筛选和复制询盘话术。但这些交互发生在在线页面中，不需要下载 HTML 文件。

### 5. API 边界

建议 API：

```text
POST /api/v1/research-tasks/{task_uuid}/report-shares
GET /api/v1/research-tasks/{task_uuid}/report-shares
POST /api/v1/report-shares/{share_uuid}/revoke
GET /api/v1/report-shares/{share_token}
```

`POST /report-shares` 创建新分享快照；同一任务允许多条分享记录，便于重新运行后创建新版分享。`revoke` 将分享置为 `revoked`，公开读取返回 404 或不可访问错误。由于当前 MVP 尚未实现账号体系，第一版先沿用现有无登录 API 边界；未来引入用户后，应把创建、列表和撤销限制到任务 owner。

## 迁移计划

1. 新增数据库迁移，创建 `report_shares` 表，包含内部 `id`、公开 `uuid`、`research_task_id`、`share_token`、`title`、`snapshot`、`status`、`revoked_at`、时间戳和软删除字段。
2. 新增 `report_sharing` 后端模块，实现快照构建、分享创建、任务内列表、撤销和公开读取服务。
3. 新增报告分享 API routes，并挂载到 v1 router。
4. 前端 API client 新增分享类型、创建分享、读取任务分享、撤销分享和公开读取分享函数。
5. 报告页新增分享组件，支持生成链接、复制链接、打开分享页和展示失败状态。
6. 新增公开分享报告页面，使用只读阅读型布局渲染快照。
7. 补齐后端、前端契约测试和 OpenSpec 校验。

回滚时可以隐藏报告页分享入口并停止公开分享路由；已创建分享记录可保留为历史数据，也可以通过状态统一置为 `revoked`。

## 风险 / 取舍

- [风险] 分享链接被转发后任何持有链接的人都能访问 -> 缓解：使用不可猜测 token，明确分享为公开链接；后续可增加过期时间或访问密码。
- [风险] 任务重新运行后用户以为旧分享同步更新 -> 缓解：文案表达为“分享快照”，重新运行后需要生成新的分享链接。
- [风险] 分享页暴露内部调试信息 -> 缓解：快照构建白名单字段，测试覆盖不暴露内部 ID、run ID、trace URL 和错误堆栈。
- [风险] 分享页阅读体验过于像后台页面 -> 缓解：使用轻量阅读型布局，保留目录、摘要、商机卡片和来源区块，避免复杂操作。
- [风险] 当前无账号体系导致撤销接口缺少真实权限边界 -> 缓解：第一版按 MVP 现状实现，设计中明确未来引入 owner 校验。

## 开放问题

- 第一版是否需要分享过期时间。建议默认不过期但支持撤销；过期策略和访问密码留给后续分享安全增强切片。
