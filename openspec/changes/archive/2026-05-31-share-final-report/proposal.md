## 为什么

当前 MVP 已经能在报告页阅读完整商机研究结果，但演示结束后用户更需要一个可以直接发给他人的在线报告链接，而不是下载文件再转发。

本 change 补齐最终报告分享能力，让用户为某次已完成研究生成可在线浏览的只读分享页，并把报告作为一个稳定快照交付出去。

## 变更内容

- 为已完成且已有商机结果的研究任务生成分享链接。
- 分享链接打开后展示在线只读报告页，包含推荐排序、商机详情摘要、需求洞察、货源候选、竞品参考、验证预算、风险复核、行动计划和来源线索。
- 分享报告保存为创建分享时的快照；任务后续重新运行不会悄悄改变旧分享页内容。
- 新增分享记录元数据，保存公开分享 UUID、不可猜测分享 token、研究任务关联、报告快照、状态、创建时间和可选撤销时间。
- 提供创建分享、查看任务分享记录、撤销分享和通过 token 读取公开报告的 API；API 不暴露内部自增 ID、run ID、trace URL 或调试字段。
- 在最终报告页提供“生成分享链接 / 复制链接 / 打开分享页”入口；分享失败不影响页面内报告继续阅读。
- 非目标：不导出 HTML 文件、不生成 PDF/Markdown/DOCX、不做对象存储文件产物、不做永久公开索引页、不做访问密码或团队权限管理。

## 能力

### 新增能力

- `report-sharing`: 定义最终报告在线分享链接的创建、快照、撤销、公开读取和前端在线浏览体验。

### 修改能力

- 无。

## 影响

- 后端：新增报告分享模块、数据库迁移、schema、repository、service 和 API routes；复用现有报告数据来源生成分享快照。
- 前端：在报告页新增分享操作与复制链接状态；新增公开分享报告路由和只读报告视图。
- 数据安全：公开分享页只暴露报告阅读需要的数据，不暴露内部 ID、任务运行调试信息、trace 链接或后台错误细节。
- 测试：新增后端分享、快照稳定、撤销、公开读取和前端契约测试。
