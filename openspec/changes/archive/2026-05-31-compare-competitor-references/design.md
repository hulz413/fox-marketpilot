## 背景

MarketPilot 现在已经完成基础商机推荐、公开来源收集、来源透明度、需求洞察和货源候选。用户可以看到“为什么可能有需求”和“可以从哪里继续找货”，但还缺少“市场上类似产品长什么样、常见售价和卖点是什么、这个方向是否容易同质化、应该从哪里做差异化”的结构化判断。

`compare-competitor-references` 是货源候选之后的 P1 质量增强切片。它复用当前研究任务、商机、来源和 Agent 阶段事件能力，为每个商机生成可读、可追踪、仍需验证的竞品参考，不改变 P0 基础商机生成边界。

## 目标 / 非目标

**目标：**

- 为每个商机生成并保存至少 2 个结构化竞品/类似产品参考。
- 覆盖类似产品示例、常见售价区间、常见卖点、同质化程度和差异化切入点。
- 优先关联现有 `competitor` 和 `general` 来源线索，缺少来源时仍生成谨慎的待验证竞品参考。
- 在商机详情展示完整竞品参考，在基础报告展示摘要。
- 为竞品参考生成阶段记录 Agent 运行事件、日志和 LangSmith trace 元数据。
- 本地和测试环境提供确定性回退，自动化测试不依赖外部 LLM、搜索或网络服务。

**非目标：**

- 不确认竞品销量、投放数据、真实成交价、库存或平台排名。
- 不抓取登录后平台数据，不绕过平台访问限制。
- 不做利润预算、风险复核或完整行动计划；这些由后续 P1 切片处理。
- 不让竞品参考反向修改商机排序、风险等级、价格带或基础推荐理由。
- 不引入 RAG、向量检索、LangSmith 数据集或自动评测；这些留给 `evaluate-rag-quality`。
- 不把竞品参考表达为“已全面核验竞品”“价格已确认”“市场已证明”或等价结论。

## 技术决策

### 1. 新增 `competitor_references` 独立模型

竞品参考作为商机结果的 P1 增强数据保存，不直接扩展 `opportunities` 表。建议新增 `competitor_references` 表：

- `id`: 内部自增主键。
- `uuid`: 对外公开 UUID。
- `research_task_id`: 内部外键，关联研究任务。
- `opportunity_id`: 内部外键，关联商机。
- `rank`: 同一商机下的参考排序。
- `reference_name`: 类似产品、竞品方向或参考商品名称。
- `reference_market`: 可能出现该类产品的公开平台、渠道或内容场景。
- `price_range`: 常见售价区间或观察到的参考价位。
- `common_selling_points`: JSON 字符串列表，常见卖点或内容表达方式。
- `homogenization_level`: `low`、`medium` 或 `high`，表达同质化程度。
- `differentiation_angles`: JSON 字符串列表，差异化切入点。
- `reference_note`: 为什么该参考与当前商机相关的说明。
- `source_status`: `linked`、`no_sources` 或 `fallback`。
- `created_at`、`updated_at`、`deleted_at`。

该表遵循项目数据库通用字段规范；默认读取只返回 `deleted_at` 为空的记录。重新运行研究任务并成功生成新商机后，旧商机关联的竞品参考应被软删除或从默认读取中排除。

替代方案：把竞品参考塞进 `opportunities` 的 JSON 字段。这样能少建表，但会让 P0 基础推荐和 P1 分析结果耦合，也不利于后续预算、风险和行动计划复用竞品参考。

### 2. 使用关联表连接竞品参考和研究来源

竞品参考需要说明哪些公开线索可作为初步参考。建议新增 `competitor_reference_sources` 表：

- `id`: 内部自增主键。
- `uuid`: 对外公开 UUID。
- `competitor_reference_id`: 内部外键，关联竞品参考。
- `research_source_id`: 内部外键，关联研究来源。
- `relevance_note`: 该来源与竞品参考的关联说明。
- `created_at`、`updated_at`、`deleted_at`。

API 返回时暴露来源 UUID、标题、URL、摘要、支撑强度和关联说明，不暴露内部自增 ID。这样保持来源数据层稳定，也能让后续预算或风险切片复用同一竞品参考及其来源关系。

替代方案：在竞品参考表中保存来源 UUID 的 JSON 数组。实现更快，但缺少关系完整性，也不利于按来源反查分析结论。

### 3. 竞品参考生成放在货源候选之后

LangGraph 节点顺序调整为：

```text
normalize_intake
  -> generate_opportunities
  -> validate_results
  -> persist_results
  -> collect_research_sources
  -> generate_demand_insights
  -> generate_supply_candidates
  -> generate_competitor_references
  -> END
```

生成节点读取当前任务的未软删除商机、来源、可用需求洞察和可用货源候选。每个商机优先使用与该商机关联且类型为 `competitor` 或 `general` 的来源；如果没有专属来源，可以使用任务级 `competitor` 或 `general` 来源；仍没有来源时，基于任务输入、商机字段和货源候选生成谨慎的待验证竞品参考，并将 `source_status` 标记为 `no_sources` 或 `fallback`。

替代方案：在商机生成前先做竞品搜索，再生成推荐。这个路径更接近完整研究 Agent，但会改变当前“先跑通基础推荐，再补强证据”的边界和失败模式，范围过大。

### 4. 使用单次批量结构化 LLM 调用

单个任务最多 5 个商机，竞品参考适合用一次批量 LLM 调用生成。输入包括任务上下文、商机基础字段、裁剪后的竞品来源摘要、可用需求洞察摘要和货源候选摘要，输出为按商机 UUID 映射的 JSON。后端使用 Pydantic 校验字段、数量、排序、文本长度和谨慎表达；校验失败可重试一次，最终失败时记录阶段失败。

本地和测试环境缺少 LLM key 时，使用确定性回退基于商机字段和来源摘要生成稳定中文竞品参考。生产环境缺少 LLM key 时按配置错误处理，但竞品参考失败不影响基础结果。

替代方案：每个商机单独调用 LLM。隔离性更好，但成本和延迟更高，且 MVP 每次只有 3-5 个商机，批量更合适。

### 5. 竞品参考失败不破坏基础结果

竞品参考是 P1 增强能力。若商机生成、校验或持久化失败，任务仍按现有核心逻辑失败；若竞品参考生成失败，任务保留 `completed`，已生成商机、来源、需求洞察和货源候选仍可读取，`generate_competitor_references` 阶段事件记录为失败，竞品参考 API 返回空结果或空状态。

这样能避免 LLM 波动或来源不足破坏已完成的 P0 演示闭环，同时让演示和排障能通过阶段事件看到竞品参考缺失原因。

替代方案：竞品参考失败即整个研究任务失败。这样能保证完成任务都有竞品参考，但会让 P1 增强能力影响核心闭环稳定性。

### 6. 前端只在结果页面内部展示

竞品参考不新增一级导航。商机详情展示完整竞品参考列表，包括参考名称、参考市场、售价区间、常见卖点、同质化程度、差异化切入点和关联来源。基础报告展示每个商机的竞品参考摘要和进入详情的入口。缺少竞品参考、来源为空或读取失败时，页面保留基础商机内容，并展示中文非阻塞空状态。

文案继续使用“类似产品参考”“公开线索”“可能”“待验证”“差异化切入点”等谨慎表达，不使用“竞品已全面核验”“售价已确认”“市场已证明”等结论式表达。

## 迁移计划

1. 新增数据库迁移，创建 `competitor_references` 和 `competitor_reference_sources`，包含 UUID、时间戳和软删除字段。
2. 新增后端模型、仓储、服务和 API，先保证空结果和 fallback 可用。
3. 将 `generate_competitor_references` 接入研究任务阶段枚举、LangGraph 节点和前端阶段文案。
4. 增加前端读取函数和展示组件，接入商机详情与基础报告。
5. 补齐后端、前端和 OpenSpec 校验后再进入实现完成状态。

回滚时可以停止调用新节点并隐藏前端面板；新表作为 P1 增强数据保留，不影响已有基础商机、来源、需求洞察或货源候选读取。

## 风险 / 取舍

- [风险] 用户误以为竞品参考已经完成市场核验 -> 缓解：规格、生成提示和 UI 均要求谨慎表达，并展示来源状态。
- [风险] 公开来源不足导致竞品参考泛化 -> 缓解：允许 `no_sources` 状态，明确参考是基于任务输入和商机字段的待验证方向。
- [风险] 新增 LLM 节点增加耗时 -> 缓解：单次批量生成，限制输入来源数量和文本长度。
- [风险] 竞品参考失败影响演示节奏 -> 缓解：非阻塞失败，保留基础结果并记录阶段事件。
- [风险] 重新运行后旧参考污染新结果 -> 缓解：按任务或商机软删除旧参考，默认读取只返回当前未软删除数据。

## 开放问题

- 第一版是否需要展示具体外链商品名称，还是只展示类似产品方向。建议第一版允许来源链接作为公开参考，但不把单个链接包装成已确认竞品。
