## 为什么

当前 MarketPilot 已经能生成基础商机、收集公开来源、解释需求洞察并给出货源候选，但用户还缺少“市场上是否已有类似产品、常见售价和卖点是什么、我能从哪里差异化切入”的判断。竞品参考是连接货源候选和后续利润预算、风险分析、行动计划的关键 P1 切片，适合作为下一步补强。

## 变更内容

- 为每个已生成商机补充竞品/类似产品参考，覆盖类似产品示例、常见售价区间、常见卖点、同质化程度和差异化切入点。
- 优先使用现有 `competitor` 或 `general` 来源线索，并在来源不足时生成谨慎的待验证竞品参考。
- 在研究运行完成来源收集、需求洞察和货源候选后生成竞品参考，并保存为可读取的结构化结果。
- 在商机详情和基础报告中展示竞品参考摘要、来源状态、差异化建议和缺少来源时的非阻塞说明。
- 提供本地和测试环境确定性回退，确保自动化测试不依赖外部 LLM、搜索或网络服务。

## 能力

### 新增能力

- `competitor-references`: 定义每个商机的竞品参考生成、持久化、读取、来源关联、前端展示和非阻塞失败契约。

### 修改能力

- `opportunity-research`: 研究运行在货源候选之后生成竞品参考，并记录对应阶段事件；竞品参考失败不破坏已完成的基础商机结果。
- `research-progress`: 进度页需要展示竞品参考生成阶段和对应的谨慎中文说明。
- `agent-run-observability`: Agent 阶段历史和 trace 需要覆盖竞品参考生成阶段，便于解释竞品参考缺失或生成失败。

## 影响

- 后端新增竞品参考领域模型、迁移、模式、仓储、服务和读取 API。
- 数据库新增 `competitor_references` 和 `competitor_reference_sources`，归属关系由 `research_task_id` 和 `opportunity_id` 表达。
- LangGraph 研究流程新增竞品参考生成阶段，并复用已有研究任务、商机、来源、需求洞察、货源候选和 Agent 运行事件能力。
- LLM 生成逻辑增加竞品参考结构化输出；本地和测试环境增加确定性回退。
- 前端商机详情和基础报告增加竞品参考展示区块，并保持缺少来源或竞品参考时的非阻塞中文空状态。
- 测试覆盖竞品参考生成、读取、来源关联、重新运行替换、运行阶段事件和前端 UI 契约。
