## MODIFIED Requirements

### Requirement: 系统执行 LangGraph 轻量商机研究 Agent

系统 SHALL 使用 LangGraph 单图 Agent 执行基础商机研究，归一化用户输入、生成结构化商机、校验结果、持久化结果，在结果保存后尝试收集公开来源线索，在来源收集后尝试建立任务内 RAG 证据索引，在 RAG 索引后进入并行研究分析阶段，并行尝试生成需求洞察、货源候选和竞品参考，在三个专业分析分支完成、跳过或安全失败后汇总研究发现，在汇总后尝试生成验证预算估算，在验证预算估算后尝试生成风险复核，并在风险复核后尝试生成行动计划。

#### Scenario: 成功执行研究运行

- **WHEN** 后台研究运行开始处理任务
- **THEN** 系统将任务状态更新为 `running`
- **AND** 系统将当前阶段更新为 `generate_opportunities`
- **AND** 系统通过 LangGraph 图执行研究节点
- **AND** 系统基于任务的自然语言需求和关键研究条件生成 3-5 个商机结果
- **AND** 每个商机结果满足 `opportunity-results` 能力定义的必填字段
- **AND** 系统持久化商机结果
- **AND** 系统进入 `collect_research_sources` 阶段并尝试收集公开来源线索
- **AND** 系统进入 `index_rag_evidence` 阶段并尝试建立任务内 RAG 证据索引
- **AND** 系统进入 `analyze_research` 阶段并并行启动 `generate_demand_insights`、`generate_supply_candidates` 和 `generate_competitor_references` 专业分析分支
- **AND** 系统在需求洞察分支尝试为每个商机生成需求洞察
- **AND** 系统在货源候选分支尝试为每个商机生成货源候选
- **AND** 系统在竞品参考分支尝试为每个商机生成竞品参考
- **AND** 系统在三个专业分析分支完成、跳过或安全失败后进入 `synthesize_research_findings` 阶段
- **AND** 系统记录三个专业分析分支的结果状态、保存数量和安全失败摘要
- **AND** 系统进入 `estimate_validation_budgets` 阶段并尝试为每个商机生成验证预算估算
- **AND** 系统进入 `review_opportunity_risks` 阶段并尝试为每个商机生成风险复核
- **AND** 系统进入 `create_action_plans` 阶段并尝试为每个商机生成行动计划
- **AND** 系统将任务状态更新为 `completed`
- **AND** 系统将当前阶段更新为 `completed`

#### Scenario: 核心研究运行失败

- **WHEN** Agent 执行、LLM 调用、结果解析、结果校验或商机结果持久化失败且无法恢复
- **THEN** 系统将任务状态更新为 `failed`
- **AND** 系统将当前阶段更新为 `failed`
- **AND** 系统保存中文可理解的失败原因
- **AND** 系统不暴露敏感凭证、原始异常堆栈或内部自增 ID 给前端用户

#### Scenario: 来源收集失败不覆盖基础结果

- **WHEN** 商机结果已经成功持久化但来源收集失败或被跳过
- **THEN** 系统保留已生成的商机结果
- **AND** 系统仍可以进入并行研究分析阶段
- **AND** 系统仍可以并行尝试生成需求洞察、货源候选和竞品参考
- **AND** 系统仍可以进入研究发现汇总阶段
- **AND** 系统仍可以进入验证预算估算阶段
- **AND** 系统仍可以进入风险复核阶段
- **AND** 系统仍可以进入行动计划生成阶段
- **AND** 系统可以将任务状态更新为 `completed`
- **AND** 系统记录来源收集阶段事件或应用日志用于排障
- **AND** 系统不把外部来源收集失败伪装成商机生成失败

#### Scenario: 需求洞察失败不覆盖基础结果

- **WHEN** 商机结果已经成功持久化但需求洞察生成失败
- **THEN** 系统保留已生成的商机结果和已收集来源
- **AND** 系统仍可以并行执行或保留货源候选分支结果
- **AND** 系统仍可以并行执行或保留竞品参考分支结果
- **AND** 系统仍可以进入研究发现汇总阶段
- **AND** 系统仍可以进入验证预算估算阶段
- **AND** 系统仍可以进入风险复核阶段
- **AND** 系统仍可以进入行动计划生成阶段
- **AND** 系统可以将任务状态更新为 `completed`
- **AND** 系统记录需求洞察阶段事件或应用日志用于排障
- **AND** 系统不把需求洞察失败伪装成商机生成失败

- **THEN** 系统保留已生成的商机结果、已收集来源、已生成需求洞察和已生成货源候选
- **AND** 系统仍可以进入研究发现汇总阶段

#### Scenario: 研究发现汇总等待并行分支

- **WHEN** 系统处于 `analyze_research` 阶段
- **THEN** 系统等待 `generate_demand_insights`、`generate_supply_candidates` 和 `generate_competitor_references` 分支均进入 completed、failed 或 skipped 等可汇总状态
- **AND** 系统不在任一分支仍处于 running 状态时进入验证预算估算阶段
- **AND** 系统进入 `synthesize_research_findings` 阶段时保留每个分支的状态 metadata
- **AND** 系统不把单个增强分析分支失败当作核心研究运行失败
