## 为什么

MarketPilot 已经具备研究质量就绪检查、生成质量评测和 RAG 检索评测能力，但这些能力主要停留在内部命令、API client 和 readiness 汇总里，演示前复查仍需要开发者在多个入口之间跳转。

现在核心 MVP 闭环已经完成，下一步需要一个只面向团队的内部质量面板，把已完成研究任务、最近质量结论、过期状态和可操作复查建议放到同一个地方，降低演示准备和问题定位成本。

## 变更内容

- 新增内部质量面板入口，用于查看已完成研究任务的演示就绪状态、生成质量评测摘要和 RAG 检索评测摘要。
- 支持在内部面板中选择研究任务，读取最近一次 readiness run，并展示整体状态、检查项、关键指标、过期状态和安全错误摘要。
- 支持从内部面板触发研究质量就绪检查和生成质量评测，运行失败时展示中文可理解的非阻断错误。
- 在面板中明确区分内部质量观察与用户侧商机结论，不把评测状态表达为市场机会已被证明。
- 保持用户进度页、商机详情页、报告页和公开分享页不展示内部质量面板、评测分数、trace metadata 或内部错误细节。

## 能力

### 新增能力

- `internal-quality-panel`: 定义内部质量面板的任务选择、质量运行触发、最近结果展示、状态摘要、安全边界和用户侧隐藏要求。

### 修改能力

- 无。

## 影响

- 前端新增内部质量面板页面、任务选择视图、质量摘要组件和运行操作。
- 前端复用或扩展现有 research API client，读取 readiness 信息并触发内部质量运行。
- 后端可能新增供内部面板使用的生成质量评测 API 或聚合读取 API，复用现有 generation quality evaluation 与 readiness service。
- 不新增用户侧主流程能力，不修改研究任务生成、商机推荐、报告分享或公开分享页的数据契约。
- 不新增第三方运行时依赖。
