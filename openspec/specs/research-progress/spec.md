## Purpose

定义 MarketPilot 研究任务进度页的用户侧展示契约，让用户在任务创建、排队、运行、失败和完成后都能理解当前状态、阶段时间线和下一步操作。

## Requirements

### Requirement: 用户可以查看研究任务进度页

系统 SHALL 为单个研究任务提供用户侧进度页，集中展示任务输入摘要、任务状态、当前阶段、运行关联和下一步操作。

#### Scenario: 打开存在的研究任务进度页

- **WHEN** 用户打开一条存在且未软删除研究任务的进度页
- **THEN** 系统展示该任务的标题、自然语言需求、关键研究条件、状态和当前阶段
- **AND** 系统展示可为空的 run ID、trace URL 和失败原因
- **AND** 系统不展示研究任务或阶段事件的内部自增 ID

#### Scenario: 打开不存在的研究任务进度页

- **WHEN** 用户打开不存在或已软删除研究任务的进度页
- **THEN** 系统展示中文未找到或不可访问说明
- **AND** 系统提供返回研究任务列表的入口

#### Scenario: 任务尚未启动

- **WHEN** 用户打开状态为 `created` 的研究任务进度页
- **THEN** 系统展示需求已提交但尚未启动研究的状态
- **AND** 系统提供启动研究入口
- **AND** 系统不展示静态 demo 阶段事件作为真实运行进度

### Requirement: 进度页展示当前运行阶段时间线

系统 SHALL 在研究任务进度页展示当前运行的阶段时间线，让用户理解已完成阶段、当前阶段、来源收集阶段、需求洞察阶段、货源候选阶段、竞品参考阶段和失败阶段。

#### Scenario: 运行中任务展示阶段进度

- **WHEN** 用户打开状态为 `queued` 或 `running` 的研究任务进度页
- **THEN** 系统展示等待后台执行、整理研究需求、生成基础推荐、校验推荐结果、保存研究结果、收集公开来源、生成需求洞察、生成货源候选和生成竞品参考等主要阶段
- **AND** 已有阶段事件展示开始时间、完成时间或运行中状态
- **AND** 当前阶段以中文可理解文案突出显示

#### Scenario: 来源收集阶段展示谨慎文案

- **WHEN** 用户打开当前阶段为 `collect_research_sources` 的研究任务进度页
- **THEN** 系统展示正在收集公开来源线索的中文说明
- **AND** 文案表达为公开线索或初步参考
- **AND** 文案不宣称商机已经完成公开市场、竞品或供给核验

#### Scenario: 货源候选阶段展示谨慎文案

- **WHEN** 用户打开当前阶段为 `generate_supply_candidates` 的研究任务进度页
- **THEN** 系统展示正在生成货源候选和供应商确认问题的中文说明
- **AND** 文案表达为候选方向、初步参考或待确认信息
- **AND** 文案不宣称供应商、库存、价格或履约能力已经被确认

#### Scenario: 竞品参考阶段展示谨慎文案

- **WHEN** 用户打开当前阶段为 `generate_competitor_references` 的研究任务进度页
- **THEN** 系统展示正在生成竞品参考、常见售价和差异化切入点的中文说明
- **AND** 文案表达为类似产品参考、公开线索或待验证信息
- **AND** 文案不宣称竞品、销量、售价或市场规模已经被全面核验

#### Scenario: 完成任务展示已完成时间线

- **WHEN** 用户打开状态为 `completed` 的研究任务进度页
- **THEN** 系统展示当前 run 已记录的已完成阶段
- **AND** 已完成阶段展示耗时或完成时间
- **AND** 系统展示进入商机推荐和基础报告的入口

#### Scenario: 来源收集失败但任务完成

- **WHEN** 用户打开一条商机结果已完成但来源收集失败或被跳过的研究任务进度页
- **THEN** 系统仍展示任务为已完成或可查看结果的状态
- **AND** 系统可以在阶段时间线中展示来源收集未获得可用来源或部分失败的中文说明
- **AND** 系统不将来源收集失败误判为基础商机生成失败

#### Scenario: 货源候选失败但任务完成

- **WHEN** 用户打开一条商机结果已完成但货源候选生成失败或没有可用候选的研究任务进度页
- **THEN** 系统仍展示任务为已完成或可查看结果的状态
- **AND** 系统可以在阶段时间线中展示货源候选暂不可用或生成失败的中文说明
- **AND** 系统不将货源候选失败误判为基础商机生成失败

#### Scenario: 竞品参考失败但任务完成

- **WHEN** 用户打开一条商机结果已完成但竞品参考生成失败或没有可用参考的研究任务进度页
- **THEN** 系统仍展示任务为已完成或可查看结果的状态
- **AND** 系统可以在阶段时间线中展示竞品参考暂不可用或生成失败的中文说明
- **AND** 系统不将竞品参考失败误判为基础商机生成失败

#### Scenario: 失败任务展示失败阶段

- **WHEN** 用户打开状态为 `failed` 的研究任务进度页
- **THEN** 系统展示中文可理解的失败原因
- **AND** 如果当前 run 存在失败阶段事件，系统标记失败阶段
- **AND** 系统提供重新运行入口

#### Scenario: 阶段事件暂不可用

- **WHEN** 任务已有 run ID 但当前 run 尚未写入阶段事件
- **THEN** 系统仍展示任务状态和当前阶段
- **AND** 系统展示中文说明表示正在等待阶段事件
- **AND** 系统不将缺少阶段事件误判为任务失败

### Requirement: 进度页按任务状态刷新和停止刷新

系统 SHALL 在运行中的研究任务进度页定期刷新状态，并在任务进入终态后停止自动刷新。

#### Scenario: 运行中自动刷新

- **WHEN** 进度页读取到任务状态为 `queued` 或 `running`
- **THEN** 前端定期重新读取任务进度
- **AND** 刷新结果更新当前阶段和阶段时间线

#### Scenario: 终态停止自动刷新

- **WHEN** 进度页读取到任务状态为 `completed` 或 `failed`
- **THEN** 前端停止运行中轮询
- **AND** 页面保留最终状态、阶段时间线和可用操作

### Requirement: 进度页提供状态相关操作

系统 SHALL 根据研究任务状态提供合适的操作入口，帮助用户继续完成 P0 演示闭环。

#### Scenario: 已完成任务进入结果

- **WHEN** 用户在已完成任务的进度页查看操作区
- **THEN** 系统提供进入商机推荐的入口
- **AND** 系统提供进入基础报告的入口

#### Scenario: 失败任务重新运行

- **WHEN** 用户在失败任务的进度页点击重新运行
- **THEN** 系统启动新的研究运行
- **AND** 进度页展示新 run 的状态和阶段
- **AND** 旧 run 阶段事件不作为当前进度默认展示

#### Scenario: 有 trace URL 的任务打开 LangSmith

- **WHEN** 用户在进度页查看一条包含 trace URL 的任务
- **THEN** 系统展示打开 LangSmith trace 的外链入口
- **AND** 点击入口后在新标签打开对应 trace

#### Scenario: 无 trace URL 的任务隐藏 LangSmith 入口

- **WHEN** 用户在进度页查看一条没有 trace URL 的任务
- **THEN** 系统不展示可点击的 LangSmith trace 入口
- **AND** 进度页其他状态和操作仍正常展示

### Requirement: 进度页提供任务上下文导航

研究任务进度页 SHALL 提供同一任务下的结果导航，让用户从进度继续进入商机推荐、基础报告和来源线索。

#### Scenario: 已完成任务查看上下文导航

- **WHEN** 用户打开状态为 `completed` 的研究任务进度页
- **THEN** 页面展示当前任务的商机推荐入口
- **AND** 页面展示当前任务的基础报告入口
- **AND** 页面展示当前任务的来源线索入口或来源状态

#### Scenario: 未完成任务查看上下文导航

- **WHEN** 用户打开状态为 `created`、`queued`、`running` 或 `failed` 的研究任务进度页
- **THEN** 页面展示返回研究任务列表的入口
- **AND** 对尚不可用的商机推荐或报告入口显示为不可用状态或不展示
- **AND** 页面说明需要任务完成后才能查看结果

### Requirement: 进度页突出用户可理解的阶段摘要

研究任务进度页 SHALL 在首屏展示用户可理解的当前阶段、完成情况和下一步操作，而不是优先展示技术运行字段。

#### Scenario: 查看运行中任务摘要

- **WHEN** 用户打开状态为 `queued` 或 `running` 的进度页
- **THEN** 页面首屏展示当前阶段中文名称
- **AND** 页面展示已完成阶段数量或等价进度表达
- **AND** 页面展示系统正在进行什么和用户下一步可以做什么

#### Scenario: 查看已完成任务摘要

- **WHEN** 用户打开状态为 `completed` 的进度页
- **THEN** 页面首屏说明基础商机推荐已经生成
- **AND** 页面突出查看商机推荐作为主要下一步
- **AND** 页面仍允许用户查看阶段时间线

#### Scenario: 查看失败任务摘要

- **WHEN** 用户打开状态为 `failed` 的进度页
- **THEN** 页面首屏展示中文可理解失败原因
- **AND** 页面突出重新运行作为主要下一步
- **AND** 页面可以保留返回任务列表和查看调试信息入口

### Requirement: 进度页分离调试信息和用户决策信息

研究任务进度页 SHALL 将 run ID、trace URL、阶段事件明细和技术排障信息放入调试或运行详情区域。

#### Scenario: 查看运行详情

- **WHEN** 用户需要查看 run ID、trace URL 或阶段耗时
- **THEN** 页面提供运行详情区域
- **AND** 该区域不抢占首屏主操作
- **AND** trace URL 仅在存在时展示可点击外链

#### Scenario: tracing 未启用

- **WHEN** 当前任务没有 trace URL
- **THEN** 页面隐藏 LangSmith 外链
- **AND** 页面仍展示状态、阶段摘要和可用操作
- **AND** 页面不把缺少 trace URL 表达为用户侧错误
