## Purpose

定义 MarketPilot 为每个商机生成、持久化和展示小预算验证行动计划的能力边界，帮助用户从基础商机结果进入人工可执行的首轮验证。

## Requirements

### Requirement: 系统为每个商机生成行动计划

系统 SHALL 在研究任务已有商机结果后，为每个当前未软删除商机生成一条行动计划，用于帮助用户用小预算开始首轮验证。

#### Scenario: 为已完成分析的任务生成行动计划

- **WHEN** 研究运行已经保存商机结果、货源候选、竞品参考、验证预算估算和风险复核
- **THEN** 系统为每个当前商机生成行动计划
- **AND** 每条行动计划包含验证目标、首批验证计划、选品验证方式、内容种草角度、商品标题建议、卖点建议、供应商询盘话术和上架前检查清单
- **AND** 每条行动计划关联对应研究任务和商机

#### Scenario: 上游增强数据不足时仍生成谨慎行动建议

- **WHEN** 某个商机缺少来源线索、货源候选、竞品参考、验证预算估算或风险复核
- **THEN** 系统仍可以基于任务输入、商机字段和已有增强数据生成待验证行动建议
- **AND** 系统将行动计划标记为 `fallback` 或 `insufficient_data`
- **AND** 行动计划必须提示需要继续确认供应商、样品、预算、平台规则或目标人群反馈等关键不确定性

#### Scenario: 本地或测试环境使用确定性回退

- **WHEN** 系统运行在 local 或 test 环境且未配置 LLM API key
- **THEN** 系统使用确定性 fallback 生成每个商机的行动计划
- **AND** fallback 结果仍满足行动计划 schema
- **AND** 自动化测试不依赖外部 LLM、搜索或网络服务

### Requirement: 系统持久化行动计划

系统 SHALL 将行动计划保存为可读取的结构化结果，并在重新运行时避免旧行动计划污染当前结果。

#### Scenario: 保存行动计划字段

- **WHEN** 系统成功生成行动计划
- **THEN** 系统保存每条行动计划的公开 UUID
- **AND** 系统保存研究任务关联和商机关联
- **AND** 系统保存验证目标、首批验证计划、选品验证方式、内容种草角度、商品标题建议、卖点建议、供应商询盘话术、上架前检查清单和计划状态
- **AND** 系统保存创建时间、更新时间和可为空的删除时间

#### Scenario: 重新运行替换旧行动计划

- **WHEN** 一条已完成研究任务被重新运行并成功生成新商机结果和新行动计划
- **THEN** 系统将该任务上一轮行动计划从默认读取中排除
- **AND** 系统保存新的行动计划
- **AND** 后续读取该任务行动计划列表时默认返回最新结果

#### Scenario: 默认读取不返回软删除行动计划

- **WHEN** 用户读取某条研究任务或某个商机的行动计划
- **THEN** 系统默认只返回 `deleted_at` 为空的行动计划记录
- **AND** 系统不返回该任务旧运行中已被替换的软删除行动计划

### Requirement: 用户可以读取行动计划

系统 SHALL 提供按研究任务和按商机读取行动计划的 API，用于商机详情和基础报告展示。

#### Scenario: 读取任务下全部行动计划

- **WHEN** 用户请求一条存在且未软删除研究任务的行动计划列表
- **THEN** 系统返回该任务当前未软删除商机的行动计划
- **AND** 结果可按商机推荐排序或行动计划记录创建顺序稳定展示
- **AND** API 响应不暴露研究任务、商机或行动计划的内部自增 ID

#### Scenario: 读取单个商机的行动计划

- **WHEN** 用户请求一个存在且未软删除商机的行动计划
- **THEN** 系统返回该商机当前行动计划
- **AND** 响应包含关联研究任务 UUID 和商机 UUID
- **AND** API 响应不暴露内部自增 ID

#### Scenario: 尚无行动计划时返回空结果

- **WHEN** 用户请求一条尚未生成行动计划的任务或商机
- **THEN** 系统返回空列表或空结果
- **AND** API 响应保持成功
- **AND** 前端可以展示中文非阻塞空状态

### Requirement: 前端展示行动计划

前端 SHALL 在商机详情和基础报告中展示行动计划，让用户知道下一步如何开始首轮小预算验证。

#### Scenario: 商机详情展示完整行动计划

- **WHEN** 用户打开一个已有行动计划的商机详情页
- **THEN** 页面展示验证目标、首批验证计划、选品验证方式、内容种草角度、商品标题建议、卖点建议、供应商询盘话术和上架前检查清单
- **AND** 页面使用“验证计划”“建议先做”“待确认”或等价谨慎中文文案
- **AND** 页面说明询盘话术需要用户人工调整后使用

#### Scenario: 基础报告展示行动摘要

- **WHEN** 用户打开一个已有行动计划的基础报告
- **THEN** 页面展示每个商机的首批验证计划、至少一个内容角度和至少一个上架前检查事项
- **AND** 页面提供进入商机详情查看完整行动计划的入口
- **AND** 页面不把行动计划表达为自动执行结果

#### Scenario: 缺少行动计划时不阻断结果阅读

- **WHEN** 商机详情或基础报告没有可展示的行动计划
- **THEN** 页面仍展示基础商机、需求洞察、货源候选、竞品参考、验证预算和风险复核内容
- **AND** 页面展示中文说明表示行动计划暂不可用或仍需人工整理
- **AND** 页面不把缺少行动计划误判为任务失败

### Requirement: 行动计划保持人工可执行和谨慎边界

系统 SHALL 将行动计划表达为供用户人工执行和调整的建议，不得将其包装成自动执行、保证成交或保证回本的结果。

#### Scenario: 行动计划不自动触达外部平台或供应商

- **WHEN** 系统生成或展示行动计划
- **THEN** 系统不自动下单
- **AND** 系统不自动联系供应商
- **AND** 系统不自动发布内容或创建平台商品
- **AND** 系统不要求用户提供第三方平台账号或供应商账号

#### Scenario: 行动计划不做结果承诺

- **WHEN** 系统生成或展示行动计划
- **THEN** 文案不得宣称保证成交、保证回本、供应商已确认、平台审核必过或等价结论
- **AND** 文案必须表达为建议、待验证、需要确认或人工调整后使用
- **AND** 文案必须包含至少一个用户可以执行的下一步动作

#### Scenario: 行动计划覆盖核心执行要素

- **WHEN** 系统生成行动计划
- **THEN** 每个商机的行动计划至少包含选品验证方式、内容种草角度、供应商询盘话术和上架前检查清单
- **AND** 供应商询盘话术必须覆盖起订量、样品、价格、发货、包装或售后中的多个确认点
- **AND** 上架前检查清单必须覆盖质量、履约、售后、平台规则或内容表达中的多个检查点
