## Purpose

定义 MarketPilot 验证预算估算能力，让用户在查看商机详情和基础报告时理解首批验证大致需要准备多少预算，以及哪些成本、售价或起订量假设最需要继续确认。

## Requirements

### Requirement: 系统为每个商机生成验证预算估算

系统 SHALL 在研究任务已有商机结果后，为每个当前未软删除商机生成一条验证预算估算，用于粗略判断该商机是否适合小预算验证。

#### Scenario: 为已完成分析的任务生成预算估算

- **WHEN** 研究运行已经保存商机结果、货源候选和竞品参考
- **THEN** 系统为每个当前商机生成验证预算估算
- **AND** 每条估算包含预估拿货成本、预估售价、粗略毛利空间、首批验证数量、首批验证预算和关键验证假设
- **AND** 每条估算关联对应研究任务和商机

#### Scenario: 上游增强数据不足时仍生成谨慎粗算

- **WHEN** 某个商机缺少货源候选、竞品参考或直接价格线索
- **THEN** 系统仍可以基于任务输入、商机价格带和粗略利润空间生成待验证预算估算
- **AND** 系统将估算标记为 `fallback` 或 `insufficient_data`
- **AND** 估算说明必须提示需要继续确认采购价、售价或起订量

#### Scenario: 本地或测试环境使用确定性回退

- **WHEN** 系统运行在 local 或 test 环境且未配置 LLM API key
- **THEN** 系统使用确定性 fallback 生成每个商机的验证预算估算
- **AND** fallback 结果仍满足验证预算 schema
- **AND** 自动化测试不依赖外部 LLM、搜索或网络服务

### Requirement: 系统持久化验证预算估算

系统 SHALL 将验证预算估算保存为可读取的结构化结果，并在重新运行时避免旧估算污染当前结果。

#### Scenario: 保存验证预算字段

- **WHEN** 系统成功生成验证预算估算
- **THEN** 系统保存每条估算的公开 UUID
- **AND** 系统保存研究任务关联和商机关联
- **AND** 系统保存预估拿货成本、预估售价、粗略毛利空间、首批验证数量、首批验证预算、关键验证假设、估算说明和估算状态
- **AND** 系统保存创建时间、更新时间和可为空的删除时间

#### Scenario: 重新运行替换旧预算估算

- **WHEN** 一条已完成研究任务被重新运行并成功生成新商机结果和新验证预算估算
- **THEN** 系统将该任务上一轮验证预算估算从默认读取中排除
- **AND** 系统保存新的验证预算估算
- **AND** 后续读取该任务验证预算列表时默认返回最新结果

#### Scenario: 默认读取不返回软删除预算

- **WHEN** 用户读取某条研究任务或某个商机的验证预算估算
- **THEN** 系统默认只返回 `deleted_at` 为空的估算记录
- **AND** 系统不返回该任务旧运行中已被替换的软删除预算

### Requirement: 用户可以读取验证预算估算

系统 SHALL 提供按研究任务和按商机读取验证预算估算的 API，用于商机详情和基础报告展示。

#### Scenario: 读取任务下全部验证预算估算

- **WHEN** 用户请求一条存在且未软删除研究任务的验证预算估算列表
- **THEN** 系统返回该任务当前未软删除商机的验证预算估算
- **AND** 结果可按商机推荐排序或预算记录创建顺序稳定展示
- **AND** API 响应不暴露研究任务、商机或预算估算的内部自增 ID

#### Scenario: 读取单个商机的验证预算估算

- **WHEN** 用户请求一个存在且未软删除商机的验证预算估算
- **THEN** 系统返回该商机当前验证预算估算
- **AND** 响应包含关联研究任务 UUID 和商机 UUID
- **AND** API 响应不暴露内部自增 ID

#### Scenario: 尚无预算估算时返回空结果

- **WHEN** 用户请求一条尚未生成验证预算估算的任务或商机
- **THEN** 系统返回空列表或空结果
- **AND** API 响应保持成功
- **AND** 前端可以展示中文非阻塞空状态

### Requirement: 前端展示验证预算估算

前端 SHALL 在商机详情和基础报告中展示验证预算估算，让用户理解首批验证大致需要准备多少预算以及哪些假设最需要确认。

#### Scenario: 商机详情展示完整预算估算

- **WHEN** 用户打开一个已有验证预算估算的商机详情页
- **THEN** 页面展示预估拿货成本、预估售价、粗略毛利空间、首批验证数量和首批验证预算
- **AND** 页面展示关键验证假设和估算说明
- **AND** 页面使用“粗略估算”“待验证假设”或等价谨慎中文文案

#### Scenario: 基础报告展示预算摘要

- **WHEN** 用户打开一个已有验证预算估算的基础报告
- **THEN** 页面展示每个商机的首批验证预算摘要
- **AND** 页面展示至少一个关键验证假设
- **AND** 页面提供进入商机详情查看完整预算估算的入口

#### Scenario: 缺少预算估算时不阻断结果阅读

- **WHEN** 商机详情或基础报告没有可展示的验证预算估算
- **THEN** 页面仍展示基础商机、需求洞察、货源候选和竞品参考内容
- **AND** 页面展示中文说明表示验证预算暂不可用或仍需确认
- **AND** 页面不把缺少验证预算误判为任务失败

### Requirement: 验证预算估算保持粗略谨慎

系统 SHALL 将验证预算表达为待验证粗算和关键假设，不得将其包装成精确财务结论或已确认利润。

#### Scenario: 预算估算不宣称精确财务结果

- **WHEN** 系统生成或展示验证预算估算
- **THEN** 文案不得宣称利润已确认、保证回本、确定毛利、真实成交价已确认或等价结论
- **AND** 文案必须表达为粗略估算、初步参考、待验证或需要确认
- **AND** 文案必须包含至少一个需要优先验证的关键假设

#### Scenario: 估算字段可表达区间

- **WHEN** 系统无法确认单一成本、售价、毛利或预算数值
- **THEN** 系统可以使用区间或文本说明表达估算结果
- **AND** 系统不为了填充字段而伪造精确数值
- **AND** 前端按文本展示估算字段而不要求进行精确计算
