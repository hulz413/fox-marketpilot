## Purpose

定义 MarketPilot 研究任务的创建、读取、列表展示和初始状态契约，作为后续研究执行、进度观测、商机结果和报告生成的业务锚点。

## Requirements

### Requirement: 用户可以创建商机研究任务

系统 SHALL 允许用户提交自然语言商机需求和关键研究条件，并创建一条持久化的研究任务。

#### Scenario: 使用自然语言需求创建任务

- **WHEN** 用户提交“预算 5000 元以内，从 1688 找适合小红书种草的产品，不做食品和电子产品”这类自然语言需求
- **THEN** 系统创建一条研究任务
- **AND** 任务包含唯一任务 UUID
- **AND** 任务保存用户提交的自然语言需求
- **AND** 任务标题可用于任务列表展示

#### Scenario: 保存关键研究条件

- **WHEN** 用户提交预算、目标渠道、偏好品类、排除品类、目标人群、期望利润、供给来源偏好或其他限制条件
- **THEN** 系统将这些条件保存到研究任务
- **AND** 后续读取任务时可以看到这些已提交条件

#### Scenario: 拒绝缺少核心需求的提交

- **WHEN** 用户提交的自然语言需求为空
- **THEN** 系统不创建研究任务
- **AND** 系统返回可理解的校验错误

### Requirement: 研究任务具有可衔接后续执行的初始状态

系统 SHALL 为新创建的研究任务设置清晰的初始状态和阶段，并为后续 Agent 执行与观测能力预留关联字段。

#### Scenario: 创建后的初始状态

- **WHEN** 系统成功创建研究任务
- **THEN** 任务状态为 `created`
- **AND** 当前阶段为 `intake`
- **AND** 任务包含创建时间和更新时间
- **AND** 任务包含为空的删除时间

#### Scenario: 创建任务不执行 Agent

- **WHEN** 系统成功创建研究任务
- **THEN** 系统不调用 LLM
- **AND** 系统不执行 LangGraph
- **AND** 系统不投递 Celery 研究任务
- **AND** 系统不生成商机推荐或最终报告

#### Scenario: 预留运行关联字段

- **WHEN** 系统返回新创建的研究任务
- **THEN** 任务包含可为空的运行 ID 字段
- **AND** 任务包含可为空的 trace ID 字段
- **AND** 任务包含可为空的失败原因字段

### Requirement: 研究任务可以进入执行生命周期

系统 SHALL 支持研究任务在创建后进入基础商机研究执行生命周期，同时保持创建任务不自动执行 Agent 的既有行为。

#### Scenario: 任务创建后仍不自动执行

- **WHEN** 用户成功创建研究任务
- **THEN** 任务状态仍为 `created`
- **AND** 当前阶段仍为 `intake`
- **AND** 系统不因创建动作自动调用 LLM
- **AND** 系统不因创建动作自动生成商机结果

#### Scenario: 任务启动后状态更新

- **WHEN** 用户启动研究任务运行
- **THEN** 任务状态可以从 `created` 进入 `queued`
- **AND** 后台执行开始后任务状态可以进入 `running`
- **AND** 执行成功后任务状态可以进入 `completed`
- **AND** 执行失败后任务状态可以进入 `failed`

#### Scenario: 任务读取返回执行状态

- **WHEN** 用户读取任务列表或单个任务
- **THEN** 系统返回任务当前状态
- **AND** 系统返回任务当前阶段
- **AND** 系统返回可为空的运行 ID、trace ID 和失败原因

### Requirement: 已完成研究任务关联商机结果

系统 SHALL 让完成基础商机研究的任务可以被用于查找对应商机推荐结果。

#### Scenario: 完成任务可以进入商机结果

- **WHEN** 一条研究任务状态为 `completed`
- **THEN** 系统能通过该任务 UUID 找到该任务的商机结果列表
- **AND** 前端任务列表可以提供查看商机推荐的入口

#### Scenario: 失败任务保留重试入口信息

- **WHEN** 一条研究任务状态为 `failed`
- **THEN** 系统保留失败原因
- **AND** 前端任务列表或任务详情可以提供重新运行入口

### Requirement: 用户可以查看研究任务列表

系统 SHALL 提供研究任务列表，让用户看到已经创建的商机研究任务及其当前状态。

#### Scenario: 查看已有研究任务

- **WHEN** 用户打开研究任务列表
- **THEN** 系统展示已创建的研究任务
- **AND** 每条任务至少展示任务标题、状态、当前阶段、创建时间和操作入口
- **AND** 任务按创建时间倒序排列
- **AND** 系统不展示已软删除的任务

#### Scenario: 创建后出现在任务列表

- **WHEN** 用户成功创建一条研究任务
- **THEN** 用户可以在研究任务列表看到该任务
- **AND** 列表中的任务信息与创建时提交的需求一致

#### Scenario: 没有研究任务时展示空状态

- **WHEN** 系统中没有研究任务
- **THEN** 研究任务列表展示空状态
- **AND** 空状态提供新建研究入口
- **AND** 空状态保留中文示例研究需求

### Requirement: 用户可以读取单个研究任务

系统 SHALL 允许用户通过任务 UUID 读取单个研究任务，为后续任务进度页、重新运行和结果关联提供稳定入口。

#### Scenario: 读取存在的任务

- **WHEN** 用户请求一个存在的研究任务 UUID
- **THEN** 系统返回该任务的 UUID、标题、自然语言需求、关键研究条件、状态、当前阶段、创建时间、更新时间和删除时间

#### Scenario: 读取不存在的任务

- **WHEN** 用户请求一个不存在的研究任务 UUID
- **THEN** 系统返回未找到错误
- **AND** 系统不返回其他任务的数据

#### Scenario: 不读取已软删除任务

- **WHEN** 用户请求一个已软删除的研究任务 UUID
- **THEN** 系统返回未找到错误
- **AND** 系统不返回该任务的数据

### Requirement: 新建研究页面提交真实任务

前端 SHALL 将新建研究页面接入真实任务创建 API，而不是只跳转到静态演示页面。

#### Scenario: 成功提交研究需求

- **WHEN** 用户在新建研究页面填写研究需求并点击创建任务
- **THEN** 前端调用研究任务创建 API
- **AND** 创建成功后用户进入研究任务列表或新创建任务的入口
- **AND** 用户可以看到刚创建的任务

#### Scenario: 显示提交校验错误

- **WHEN** 用户在新建研究页面提交无效输入
- **THEN** 前端展示可理解的中文错误信息
- **AND** 用户仍停留在新建研究页面修改输入
