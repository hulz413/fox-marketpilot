## Purpose

定义 MarketPilot 研究任务的创建、读取、列表展示和初始状态契约，作为后续研究执行、进度观测、商机结果和报告生成的业务锚点。

## Requirements

### Requirement: 用户可以创建商机研究任务

系统 SHALL 允许用户提交自然语言商机需求和关键研究条件，并创建一条持久化的研究任务。

#### Scenario: 使用自然语言需求创建任务

- **WHEN** 用户提交“预算 5000 元以内，从 1688 找适合小红书种草的产品，不做食品和电子产品”这类自然语言需求
- **THEN** 系统创建一条研究任务
- **AND** 任务包含唯一任务 UUID
- **AND** 任务保存用户提交的自然语言需求
- **AND** 任务标题可用于任务列表展示

#### Scenario: 保存关键研究条件

- **WHEN** 用户提交预算、目标渠道、偏好品类、排除品类、目标人群、期望利润、供给来源偏好或其他限制条件
- **THEN** 系统将这些条件保存到研究任务
- **AND** 后续读取任务时可以看到这些已提交条件

#### Scenario: 拒绝缺少核心需求的提交

- **WHEN** 用户提交的自然语言需求为空
- **THEN** 系统不创建研究任务
- **AND** 系统返回可理解的校验错误

### Requirement: 研究任务具有可衔接后续执行的初始状态

系统 SHALL 为新创建的研究任务设置清晰的初始状态和阶段，并为后续 Agent 执行与观测能力预留关联字段。

#### Scenario: 创建后的初始状态

- **WHEN** 系统成功创建研究任务
- **THEN** 任务状态为 `created`
- **AND** 当前阶段为 `intake`
- **AND** 任务包含创建时间和更新时间
- **AND** 任务包含为空的删除时间

#### Scenario: 创建任务不执行 Agent

- **WHEN** 系统成功创建研究任务
- **THEN** 系统不调用 LLM
- **AND** 系统不执行 LangGraph
- **AND** 系统不投递 Celery 研究任务
- **AND** 系统不生成商机推荐或最终报告

#### Scenario: 预留运行关联字段

- **WHEN** 系统返回新创建的研究任务
- **THEN** 任务包含可为空的运行 ID 字段
- **AND** 任务包含可为空的 trace ID 字段
- **AND** 任务包含可为空的失败原因字段

### Requirement: 研究任务可以进入执行生命周期

系统 SHALL 支持研究任务在创建后进入基础商机研究执行生命周期，同时保持创建任务不自动执行 Agent 的既有行为。

#### Scenario: 任务创建后仍不自动执行

- **WHEN** 用户成功创建研究任务
- **THEN** 任务状态仍为 `created`
- **AND** 当前阶段仍为 `intake`
- **AND** 系统不因创建动作自动调用 LLM
- **AND** 系统不因创建动作自动生成商机结果

#### Scenario: 任务启动后状态更新

- **WHEN** 用户启动研究任务运行
- **THEN** 任务状态可以从 `created` 进入 `queued`
- **AND** 后台执行开始后任务状态可以进入 `running`
- **AND** 执行成功后任务状态可以进入 `completed`
- **AND** 执行失败后任务状态可以进入 `failed`

#### Scenario: 任务读取返回执行状态

- **WHEN** 用户读取任务列表或单个任务
- **THEN** 系统返回任务当前状态
- **AND** 系统返回任务当前阶段
- **AND** 系统返回可为空的运行 ID、trace ID 和失败原因

### Requirement: 已完成研究任务关联商机结果

系统 SHALL 让完成基础商机研究的任务可以被用于查找对应商机推荐结果。

#### Scenario: 完成任务可以进入商机结果

- **WHEN** 一条研究任务状态为 `completed`
- **THEN** 系统能通过该任务 UUID 找到该任务的商机结果列表
- **AND** 前端任务列表可以提供查看商机推荐的入口

#### Scenario: 失败任务保留重试入口信息

- **WHEN** 一条研究任务状态为 `failed`
- **THEN** 系统保留失败原因
- **AND** 前端任务列表或任务详情可以提供重新运行入口

### Requirement: 用户可以查看研究任务列表

系统 SHALL 提供研究任务列表，让用户看到已经创建的商机研究任务及其当前状态。

#### Scenario: 查看已有研究任务

- **WHEN** 用户打开研究任务列表
- **THEN** 系统展示已创建的研究任务
- **AND** 每条任务至少展示任务标题、状态、当前阶段、创建时间和操作入口
- **AND** 任务按创建时间倒序排列
- **AND** 系统不展示已软删除的任务

#### Scenario: 创建后出现在任务列表

- **WHEN** 用户成功创建一条研究任务
- **THEN** 用户可以在研究任务列表看到该任务
- **AND** 列表中的任务信息与创建时提交的需求一致

#### Scenario: 没有研究任务时展示空状态

- **WHEN** 系统中没有研究任务
- **THEN** 研究任务列表展示空状态
- **AND** 空状态提供新建研究入口
- **AND** 空状态保留中文示例研究需求

### Requirement: 用户可以读取单个研究任务

系统 SHALL 允许用户通过任务 UUID 读取单个研究任务，为后续任务进度页、重新运行和结果关联提供稳定入口。

#### Scenario: 读取存在的任务

- **WHEN** 用户请求一个存在的研究任务 UUID
- **THEN** 系统返回该任务的 UUID、标题、自然语言需求、关键研究条件、状态、当前阶段、创建时间、更新时间和删除时间

#### Scenario: 读取不存在的任务

- **WHEN** 用户请求一个不存在的研究任务 UUID
- **THEN** 系统返回未找到错误
- **AND** 系统不返回其他任务的数据

#### Scenario: 不读取已软删除任务

- **WHEN** 用户请求一个已软删除的研究任务 UUID
- **THEN** 系统返回未找到错误
- **AND** 系统不返回该任务的数据

### Requirement: 新建研究页面提交真实任务

前端 SHALL 将新建研究页面接入真实任务创建 API，而不是只跳转到静态演示页面。

#### Scenario: 成功提交研究需求

- **WHEN** 用户在新建研究页面填写研究需求并点击创建任务
- **THEN** 前端调用研究任务创建 API
- **AND** 创建成功后用户进入研究任务列表或新创建任务的入口
- **AND** 用户可以看到刚创建的任务

#### Scenario: 显示提交校验错误

- **WHEN** 用户在新建研究页面提交无效输入
- **THEN** 前端展示可理解的中文错误信息
- **AND** 用户仍停留在新建研究页面修改输入

### Requirement: 研究任务可以关联 Agent trace

系统 SHALL 让研究任务记录能够关联最近一次基础商机研究运行的 run ID、trace ID 和失败摘要。

#### Scenario: 读取已追踪运行的任务

- **WHEN** 用户读取一条已经启动基础商机研究运行的任务
- **THEN** 系统返回该任务当前运行 ID
- **AND** 如果该运行已成功创建 LangSmith trace，系统返回对应 trace ID
- **AND** 如果系统能构造对应 LangSmith trace 页面地址，系统返回可空 trace URL
- **AND** 系统不返回内部自增 ID

#### Scenario: 重新运行刷新运行关联

- **WHEN** 用户重新运行一条已完成或已失败的研究任务
- **THEN** 系统为新运行生成新的运行 ID
- **AND** 系统清空上一轮失败原因
- **AND** 系统将 trace ID 更新为新运行的 trace ID，或在新运行尚未创建 trace 时保持为空

#### Scenario: tracing 未启用时任务仍可读取

- **WHEN** 用户读取一条在 LangSmith tracing 未启用环境中运行的任务
- **THEN** 系统仍返回任务状态、当前阶段、运行 ID 和失败原因
- **AND** trace ID 可以为空
- **AND** trace URL 可以为空
- **AND** API 响应保持成功且不暴露 tracing 初始化细节

### Requirement: 失败任务保留可展示失败摘要

系统 SHALL 在研究任务失败时保留中文可展示失败摘要，并让开发者能通过运行关联字段继续排障。

#### Scenario: 读取失败任务

- **WHEN** 用户读取一条失败的研究任务
- **THEN** 系统返回状态 `failed`
- **AND** 系统返回中文可理解的失败原因
- **AND** 系统返回当前运行 ID
- **AND** 如果失败运行创建过 trace，系统返回 trace ID
- **AND** 失败原因不包含敏感凭证、原始异常堆栈或内部自增 ID

### Requirement: 研究任务列表提供进度入口

系统 SHALL 让研究任务列表可以进入单个任务的进度页，使用户从列表继续查看运行状态、失败原因和下一步操作。

#### Scenario: 运行中任务进入进度页

- **WHEN** 用户在研究任务列表查看状态为 `queued` 或 `running` 的任务
- **THEN** 前端展示进入任务进度页的入口
- **AND** 该入口使用研究任务 UUID
- **AND** 前端不要求用户先进入商机列表或报告页

#### Scenario: 失败任务进入进度页

- **WHEN** 用户在研究任务列表查看状态为 `failed` 的任务
- **THEN** 前端展示进入任务进度页的入口
- **AND** 进度页可以展示该任务的失败原因和重新运行入口

#### Scenario: 已完成任务保留结果入口

- **WHEN** 用户在研究任务列表查看状态为 `completed` 的任务
- **THEN** 前端仍提供商机推荐入口
- **AND** 前端仍提供基础报告入口
- **AND** 前端可以提供进度页入口用于查看本次运行阶段历史

### Requirement: 新建研究成功后进入任务进度页

前端 SHALL 在用户创建并启动研究任务后进入新任务的进度页，而不是只返回任务列表。

#### Scenario: 创建并启动后跳转进度页

- **WHEN** 用户在新建研究页面提交有效需求并成功启动研究运行
- **THEN** 前端跳转到该研究任务的进度页
- **AND** 进度页展示该任务的运行中状态或最新阶段
- **AND** 用户无需从任务列表再次寻找刚创建的任务

#### Scenario: 启动失败后保留错误信息

- **WHEN** 用户创建任务成功但启动研究运行失败
- **THEN** 前端展示中文可理解的失败信息
- **AND** 用户可以进入任务进度页查看任务状态或重新运行

### Requirement: 任务进度页可以启动或重新运行研究任务

系统 SHALL 允许用户从研究任务进度页对可运行任务启动或重新运行基础商机研究。

#### Scenario: 从已创建任务启动研究

- **WHEN** 用户在状态为 `created` 的任务进度页点击启动研究
- **THEN** 系统为该任务生成 run ID
- **AND** 系统将任务状态更新为 `queued`
- **AND** 进度页展示新的运行状态

#### Scenario: 从失败任务重新运行研究

- **WHEN** 用户在状态为 `failed` 的任务进度页点击重新运行
- **THEN** 系统为该任务生成新的 run ID
- **AND** 系统清空上一轮失败原因
- **AND** 系统将任务状态更新为 `queued`
- **AND** 进度页展示新 run 的进度

#### Scenario: 避免重复启动运行中任务

- **WHEN** 用户在状态为 `queued` 或 `running` 的任务进度页尝试启动研究
- **THEN** 系统不创建重复运行
- **AND** 前端继续展示当前 run 的进度
