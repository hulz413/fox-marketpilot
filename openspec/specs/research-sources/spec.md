## Purpose

定义 MarketPilot 基础商机研究运行的公开来源收集、持久化、读取和安全边界，为后续来源透明度、报告引用和更深度分析能力提供数据基础。

## Requirements

### Requirement: 系统持久化研究来源

系统 SHALL 将每次研究运行收集到的公开来源保存为可读取的持久化记录，并与研究任务建立稳定关联。

#### Scenario: 保存来源基础字段

- **WHEN** 来源收集阶段获得公开来源候选
- **THEN** 系统保存来源 UUID、研究任务关联、来源标题、URL、摘要、片段、来源类型、关联判断、支撑强度和收集时间
- **AND** 来源可以可选关联到单个商机
- **AND** 来源包含创建时间、更新时间和可为空的删除时间
- **AND** 系统不对外暴露来源、任务或商机的内部自增 ID

#### Scenario: 来源类型可区分判断用途

- **WHEN** 系统保存研究来源
- **THEN** 每条来源的类型为 `demand`、`supply`、`competitor`、`risk` 或 `general` 之一
- **AND** 来源类型用于表达该来源主要支持需求、货源、竞品、风险或通用背景判断

#### Scenario: 来源支撑强度可表达不确定性

- **WHEN** 系统保存研究来源
- **THEN** 每条来源的支撑强度为 `weak`、`medium` 或 `strong` 之一
- **AND** 支撑强度只表示该来源与关联判断的相关程度
- **AND** 支撑强度不表示商机已经被证明成立

#### Scenario: 重新运行替换旧来源

- **WHEN** 一条已完成研究任务被重新运行并成功完成新的商机结果和来源收集
- **THEN** 系统将该任务上一轮来源从默认读取中排除
- **AND** 系统保存新运行收集到的来源
- **AND** 后续读取该任务来源列表时默认返回最新运行的未软删除来源

### Requirement: 系统收集公开来源线索

系统 SHALL 在基础商机结果保存后，为研究任务和已生成商机收集公开来源线索。

#### Scenario: 基于任务和商机生成来源查询

- **WHEN** 基础商机结果已经成功保存
- **THEN** 系统基于研究任务输入、目标渠道、供给来源偏好、商机名称、产品方向和目标人群生成来源查询
- **AND** 查询覆盖需求、货源、竞品或风险中的至少一种判断类型
- **AND** 系统限制单次任务的查询数量和保存来源数量

#### Scenario: 使用公开搜索结果创建来源候选

- **WHEN** 系统执行来源收集
- **THEN** 系统可以使用配置的公开搜索服务获取来源标题、链接、片段和相关性分数
- **AND** 系统只使用公开可访问来源
- **AND** 系统不要求登录平台账号
- **AND** 系统不抓取私域、付费墙后或需要用户授权的数据

#### Scenario: 可选提取正文增强摘要

- **WHEN** 来源候选需要更完整的摘要且 URL 数量在系统限制内
- **THEN** 系统可以对候选 URL 执行正文提取
- **AND** 提取失败时系统仍可使用搜索片段生成来源摘要
- **AND** 提取失败不会导致基础商机结果丢失

#### Scenario: 本地或测试环境不依赖外部搜索服务

- **WHEN** 系统运行在 local 或 test 环境且未配置公开搜索 API key
- **THEN** 系统使用确定性 fallback 生成少量中文来源线索或将来源收集标记为跳过
- **AND** 自动化测试不依赖外部网络服务
- **AND** 基础商机研究任务仍可完成

### Requirement: 来源摘要保持谨慎表达

系统 SHALL 将来源表达为公开线索或初步参考，避免把来源包装成完整市场核验结论。

#### Scenario: 来源摘要使用待验证语气

- **WHEN** 系统生成来源摘要或关联判断
- **THEN** 文案使用“可能支持”“可作为初步参考”“提示存在相关信号”或等价谨慎表达
- **AND** 文案不使用“已证明”“确定有市场”“供给已确认”“竞品已全面核验”或等价表达

#### Scenario: 来源关联到具体判断

- **WHEN** 系统保存研究来源
- **THEN** 系统保存该来源关联的判断说明
- **AND** 关联判断说明可用于后续报告或详情页解释来源为什么被引用
- **AND** 关联判断说明不要求本阶段改写商机推荐理由

### Requirement: 用户可以读取研究任务来源

系统 SHALL 允许前端读取某条研究任务的研究来源列表，为后续来源透明度 UI 和报告引用提供数据。

#### Scenario: 读取任务来源列表

- **WHEN** 用户请求一条存在且未软删除研究任务的来源列表
- **THEN** 系统返回该任务未软删除的来源
- **AND** 来源包含 UUID、标题、URL、摘要、片段、来源类型、关联判断、支撑强度、可空商机 UUID 和收集时间
- **AND** 来源按来源类型、商机关联和收集时间稳定排序
- **AND** 系统不返回来源、任务或商机的内部自增 ID

#### Scenario: 读取没有来源的任务

- **WHEN** 用户请求一条尚未收集来源或来源收集被跳过的研究任务来源列表
- **THEN** 系统返回空来源列表
- **AND** API 响应保持成功
- **AND** 前端可以展示中文空状态或来源暂不可用说明

#### Scenario: 读取不存在任务的来源

- **WHEN** 用户请求一个不存在或已软删除研究任务的来源列表
- **THEN** 系统返回未找到错误
- **AND** 系统不返回其他任务的来源数据

### Requirement: 用户可以读取单个商机来源

系统 SHALL 允许前端读取某个商机关联的研究来源列表。

#### Scenario: 读取商机关联来源

- **WHEN** 用户请求一个存在且未软删除商机的来源列表
- **THEN** 系统返回关联到该商机的未软删除来源
- **AND** 每条来源包含来源 UUID、标题、URL、摘要、来源类型、关联判断、支撑强度和收集时间
- **AND** 系统不返回其他商机的专属来源

#### Scenario: 读取不存在商机的来源

- **WHEN** 用户请求一个不存在或已软删除商机的来源列表
- **THEN** 系统返回未找到错误
- **AND** 系统不返回其他商机或任务的来源数据

### Requirement: 来源收集失败不破坏基础商机闭环

系统 SHALL 将来源收集视为 P1 增强能力，使外部搜索失败不会破坏已完成的 P0 基础推荐结果。

#### Scenario: 来源收集全部失败

- **WHEN** 商机结果已经成功保存但来源搜索、提取或摘要全部失败
- **THEN** 研究任务仍可保持 `completed`
- **AND** 系统不删除已生成的商机结果
- **AND** 系统记录来源收集失败或跳过的阶段事件和应用日志
- **AND** 来源读取接口返回空列表

#### Scenario: 来源收集部分成功

- **WHEN** 来源收集阶段只成功处理部分查询或 URL
- **THEN** 系统保存成功处理的来源
- **AND** 系统记录部分失败的可观测信息
- **AND** 系统不因为部分来源失败而让研究任务进入 `failed`

#### Scenario: 来源收集错误信息保持安全

- **WHEN** 来源收集阶段发生外部 API、网络、提取或摘要错误
- **THEN** 用户可见错误摘要不包含敏感凭证、原始异常堆栈或内部自增 ID
- **AND** 开发者仍可通过日志、run ID 和阶段事件定位失败原因
